## 本版目标

在 v1.3.0 主链路之上，交付 **经营分析四能力** 的首个模块：**产能分析**（负荷描述 + 产线甘特图 + 占用详情）。库存查询、业务分析、回款分析 **保持可点、不置灰**，仍使用 v1.3 Insight 卡，待后续版本升级。

## 变更摘要（相对 v1.3.0）

| 维度 | v1.3.0 | v1.4.0 |
|------|--------|--------|
| 产能分析 | 演示 Insight 四行 KPI | **负荷描述 + 产线甘特 + Tab 订单详情** |
| 库存 / 业务 / 回款 | 演示 Insight 卡 | **不置灰**，仍 Insight 卡 |
| 欢迎区 / 技能条 | 四分析可点 | **四分析均可点**；产能本版验收 redesign |

## 主功能定义 · 产能分析（A2）

| 项 | 说明 |
|----|------|
| **做什么** | 在已选客户前提下，展示 **负荷描述**（排产至日期 + **平均负荷率**）+ **产线占用甘特图**（只读）+ 点击占用块查看 **订单占用详情**。 |
| **不做什么** | 不接真实 MES/APS 排程引擎；不支持拖拽改排、锁单、下达工序（B 端完整排程台能力不在 H5 范围）。 |
| **前置** | 顶栏已选当前客户（与 v1.3 分析类技能一致）。 |
| **完成标准** | 入口 → 对话内结果卡：负荷描述 + 可横滑甘特（首屏 1 天）+ 订单详情 Tab 可回甘特；设计标注验收完整。 |
| **数据来源** | 甘特 + 占用详情：**已有后台能力直返**（功能描述 §1.1.9）；负荷描述：**销售助手基于甘特数据计算**（§1.1.10）；H5 原型用本地演示数据模拟完整结果。 |

**入口**：欢迎区「产能分析」· 底部技能条 · 话术「产能分析」「排到哪天了」等。

## 关键交互（A5 · 纯前端模拟）

1. 用户触发产能分析 → 请求销售助手（原型读本地演示数据）。
2. 推送 **结果卡**：
   - **块 A · 负荷描述**：排产至日期、平均负荷率、负荷摘要文案（§1.1.10 **计算**）。
   - **块 B · 甘特图**：**已有能力直返**；一行一产线 + 占用块；首屏 **1 天**，横滑。
3. 点击占用块 → **订单占用详情随块一并返回** → 自动切「订单详情」Tab。
4. **回甘特**：点 **「甘特图」Tab**；甘特 **保留横滑位置**，可继续点其它占用块。
5. **当前时刻**竖线、**非工作时段**灰带、占用块色块对齐参考 ERP 甘特语义，交互只读。

## 业务规则

| 规则 | 说明 |
|------|------|
| 客户必选 | 与其它分析技能一致 |
| 只读 | 甘特图不可拖拽改排 |
| 时间轴 | **小时**粒度（无切换控件）；首屏默认 1 天；窄屏横滑 |
| 负荷结论 | 排产至日期 + 平均负荷率（今日～排产至日期，工作实际小时数÷可工作小时数；§1.1.10） |
| 四分析入口 | 库存/业务/回款 **不置灰** |

## 不在本版

- 库存查询 / 业务分析 / 回款分析 **完整 redesign**（仍 Insight 卡）
- 甘特拖拽、锁单、预排下达
- 按客户过滤产线（若后台无此维度则展示工厂级总览）
