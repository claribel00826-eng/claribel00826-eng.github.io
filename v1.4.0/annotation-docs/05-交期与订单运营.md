## 二、交期评审（§1.5）

### 2.1 入口 · `card-delivery-entry`

1. **内容**：标题「交期评审」、当前客户名称、客户类型标签（新/老客户）；主按钮「评估交期」。
2. **查询逻辑**：顶栏须已选客户；客户类型规则同方案速配 §1.2.1。
3. **交互逻辑**：评估交期 → 来源选择卡；技能条直达来源时可跳过本卡。

### 2.2 来源选择 · `card-delivery-source`

1. **内容**：标题「交期评审 · 选择来源」；三选项互斥——按报价单 / 按订单 / 自选商品；无可用数据时对应选项置灰并副文案说明数量。
2. **查询逻辑**：报价单 = 当前客户全部报价单；未排程订单 = 非待提交、非已完成。
3. **交互逻辑**：
   - 按报价单：1 份或会话当前报价单 → 评审表单；多份 → 选报价单。
   - 按订单：1 笔未排程 → 评审表单；多笔 → 选订单。
   - 自选商品：新客户需求引导 → 交期自选选品 → 确认选品 → 评审表单。

### 2.3 选报价单 · `card-delivery-quote-pick`

1. **内容**：列表行——报价单号、金额、模板名。
2. **查询逻辑**：当前客户全部报价单，按生成时间倒序。
3. **交互逻辑**：点选一行 → 载入报价明细 → 评审表单。

### 2.4 选订单（交期） · `card-delivery-order-pick`

1. **内容**：列表行——订单号、五类订单类型徽章、下单日期 · 金额。
2. **查询逻辑**：当前客户未排程订单，按下单日期倒序。
3. **交互逻辑**：点选一行 → 只读订单行明细 → 评审表单。

### 2.5 需求引导（自选） · `card-delivery-demand`

1. **内容**：采购需求输入；老客户展示「跳过，按最近订单推荐」。
2. **查询逻辑**：规则同 §1.2.3.1（新客户须填需求；老客户可跳过）。
3. **交互逻辑**：确认需求 / 跳过 / 对话完整需求句 → 交期自选·选品卡。

### 2.6 自选·选品 · `card-order-pick`（交期模式）

1. **内容**：标题「交期评审 · 自选商品」；推荐区 + 筛选；底部「下一步：确认选品」。
2. **查询逻辑**：推荐 / 筛选规则同 §1.2.3.2 / §1.4.5。
3. **交互逻辑**：确认选品 → 汇总明细 → 评审表单。

### 2.7 评审表单 · `sheet-delivery`

1. **内容**（自上而下）：
   - 标题「交期评审」
   - **来源摘要**（只读）：按报价单 `报价单 QT… · ¥金额` / 按订单 `订单 SO… · 状态` / 自选 `N 项 · 品名…`
   - **期望交期**：日期，默认为空
   - **工艺版本（按货品）**：明细表 `# / 品名 / 工艺版本`，每行下拉
   - **是否生成采购计划**：单选 是/否；**默认由后台带入，用户可修改**
   - 主按钮「提交评审」
   - **不含**开始时间、结束时间字段
2. **查询逻辑**：明细来自报价行 / 订单行 / 自选汇总；工艺版本候选项按货品规格从主数据读取。
3. **交互逻辑**：
   - 未选期望交期 → toast「请选择期望交期」
   - 任一行未选工艺版本 → toast「请为每项选择工艺版本」
   - 提交 → 连同用户确认后的采购计划选项调用后台评审 → 写入会话（已确认）→ 结果卡

### 2.8 评审结果 · `card-delivery`

1. **内容**（自上而下）：
   - 标题「交期评审 ·」+ **结论徽章**（按期 / 无法按时交付，后台返回）
   - **结论区**（仅「按期」）：绿色结论文案 + 灰色说明行（如「预计 … 可整单交付」；后台返回）
   - **原因区**（仅「无法按时交付」）：浅黄底原因框、**无标题**（后台返回）
   - **参数摘要**（浅灰底，**全部只读不可改**）：
     - 开始时间 / 结束时间（同一行两列，后台返回）
     - 期望交期（表单回显）
     - 工艺版本（按货品分行：品名 + 版本号）
     - 采购计划（是/否，取表单提交值；**结果卡不可修改**）
   - 操作区：主按钮 1 个；无法按时交付时附加「调整方案」
   - **不展示**「评审完成，可以生成订单。」等引导文案
2. **查询逻辑**：开始/结束时间、评审结果、无法按时交付原因均为后台返回；会话已确认评审结果。
3. **交互逻辑**：
   - 来源报价单/自选：**生成订单** → 预填待确认订单 → 下单确认
   - 来源订单：**查看订单进度** → 进度详情卡
   - 无法按时交付：**调整方案** → 方案速配
   - 交期与下单并列；本卡「生成订单」不阻塞报价/复制等主链路

## 三、复制订单（§1.6）

### 3.1 选单 · `card-order-pick`（复制模式）

1. **内容**：订单号、五类类型徽章、日期 · 金额。
2. **查询逻辑**：当前客户历史订单，按日期倒序。
3. **交互逻辑**：点选 → 复制明细确认；无历史订单仅对话提示。

### 3.2 明细确认 · `card-order-copy`

1. **内容**：标题「复制订单 · 明细确认」；来源订单号 + 类型徽章 + 日期；客户名；可点击明细提示；手风琴明细行；合计金额。
2. **查询逻辑**：来源订单明细写入待确认订单；`sourceType: copy`。
3. **交互逻辑**：点行展开/收起（同时仅一行）；改规格/数量/单价 → 重算合计；**确认下单** → 下单确认（来源「复制订单 · SOxxx」）→ 提交后状态未审核。

## 四、订单变更（§1.7）

### 4.1 选单 · `card-order-pick`（变更模式）

同复制选单；已完成订单 toast 不可变更。

### 4.2 变更表单 · `sheet-change`

1. **内容**：变更原因（下拉）、变更备注（必填）。
2. **查询逻辑**：绑定当前变更订单。
3. **交互逻辑**：备注为空 toast；提交 → 变更已提交卡；已审核订单回退销售审核。

### 4.3 变更成功 · `card-change-success`

1. **内容**：订单号、变更原因、变更备注（只读）。
2. **交互逻辑**：停留对话页，无强制跳转。

## 五、订单进度（§1.8）

### 5.1 五类订单类型（列表 / 详情共用）

| 类型 | 徽章风格 |
|------|----------|
| 未审核 | 中性灰 |
| 销售审核 | 主色描边 |
| 已审核 | 成功绿 |
| 已完成 | 弱化灰 |
| 异常 | 警告色 |

时间轴：**未审核 → 销售审核 → 已审核 → 已完成**；异常节点标红；已完成全节点打勾。

### 5.2 进度列表 · `card-order-progress-list`

1. **内容**：订单号、类型徽章、状态说明、品项摘要 · 日期、「查看详情」。
2. **查询逻辑**：当前客户全部订单，按下单日期倒序。
3. **交互逻辑**：查看详情 → 进度详情卡。

### 5.3 进度详情 · `card-order-progress-detail`

1. **内容**：订单号 + 类型徽章；状态说明；品项 · 金额；审核履约时间轴。
2. **查询逻辑**：订单进度节点（done / current / error / 时间）。
3. **交互逻辑**：入口含技能、交期结果「查看订单进度」、话术；只读无提交。

## 七、标注三栏摘要

| 标注标识 | 内容 | 查询 | 交互 |
|---------|------|------|------|
| 交期评审 · 入口 | 客户 + 评估交期 | 顶栏客户 | → 来源选择 |
| 交期评审 · 选择来源 | 三来源 + 置灰规则 | 报价单 / 未排程订单 | 报价单 / 订单 / 自选 |

| 交期评审 · 需求引导 | 需求输入 + 跳过 | 新/老客户 | → 选品 |
| 订单选单 / 交期自选·选品 | 三场景共用宿主 | 见场景 | 复制 / 变更 / 确认选品 |

| 订单变更 · 表单 | 原因 + 备注 | 绑定订单 | 提交变更 |
| 变更已提交 | 只读确认 | — | — |
| 订单进度 · 列表 | 五类状态 | 客户订单 | 详情 |
| 订单进度 · 详情 | 时间轴 | 进度节点 | 只读 |
