# 功能架构（跨版本）

> 各版本 PRD 在此基础上做 MVP / V2+ 切分；实现以对应 `v*/index.html` 为准。

## 信息架构

```
登录（可选 / 演示可跳过）
    └── 对话页 #chat
            ├── 顶栏（智能体名、企业切换）
            ├── 状态区（当前功能、当前客户）
            ├── 消息区（欢迎 / 待跟进 / 卡片 / 用户气泡）
            ├── 技能横滑条
            └── 输入区（语音 + 文字）
```

## 核心链路（MVP 已验证于 v1.1.0）

1. **待跟进**：摘要 → 列表 → 选客户 → 详情 → 下一步引导  
2. **写跟进**：意图/技能 →（缺客户则引导）→ 抽屉表单 → 提交确认  
3. **切换客户**：话术 / 弹窗列表  
4. **意图填槽**：全局槽（功能/客户/产品）+ 写跟进表单槽  

## 演示外壳（非业务）

- 根目录 `index.html`：版本索引  
- 各版本 `index.html`：`sc-demo-shell`（工具栏、可选微信模板区、`sc-app` H5 框）  

## v1.2.0 主链路（已实现于 `v1.2.0/`）

1. **方案速配**：选客户 → 选品卡 → 购物车 → 方案卡（`ctx.scheme`）  
2. **产品报价**：须已有方案 → 报价抽屉 → 报价单卡（`ctx.quote`）  
3. **确认下单**：须已有报价 → 下单抽屉 → 订单成功卡（**不要求交期**）

交期评审、复制订单等仍在技能条，与主链路并行演示。

## V2+ 扩展

复制/变更/进度/客服完整流程；服务号深链自动设客。
