## 1.5 交期评审

### 1.5.1 入口卡

**前置条件**：顶栏已选当前客户。

**出现时机**：技能条 / 欢迎区 / 话术「交期」「评估交期」等。

**内容**：

- 标题区：功能名「交期评审」、当前客户名称、客户类型标签（新客户 / 老客户）
- 主操作：**评估交期** → 进入来源选择

**字段说明**：


| 字段   | 说明                 |
| ---- | ------------------ |
| 客户名称 | 当前顶栏选中客户           |
| 客户类型 | 规则同方案速配 **§1.2.1** |


**交互**：


| 操作   | 结果                |
| ---- | ----------------- |
| 评估交期 | 展示 **来源选择卡**（三选一） |


---

### 1.5.2 来源选择卡

**标题**：交期评审 · 选择来源

**内容**（三选一，互斥）：


| 选项       | 说明               | 可用条件                     |
| -------- | ---------------- | ------------------------ |
| **按报价单** | 以历史或当前报价单明细评估交期  | 本客户至少有 1 份报价单；否则置灰       |
| **按订单**  | 以未排程销售订单明细评估交期   | 本客户至少有 **1 笔未排程订单**；否则置灰 |
| **自选商品** | 直选品 + 规格 + 数量后评估 | 始终可用                     |


**交互**：


| 操作   | 结果                                                                       |
| ---- | ------------------------------------------------------------------------ |
| 按报价单 | 仅 **1** 份报价单 → 直接进入 **评审表单**；多份 → **报价单选单卡**；会话当前报价单优先载入                 |
| 按订单  | 仅 **1** 笔可选订单 → 直接进入 **评审表单**；多笔 → **订单选单卡（交期）**                         |
| 自选商品 | 新客户须先 **需求引导**（规则同 **§1.2.3.1**）→ **交期自选·选品卡** → **下一步：确认选品** → **评审表单** |


---

### 1.5.3 报价单选单卡（按报价单 · 多报价单）

**查询逻辑**：当前客户全部报价单，按生成时间倒序。

**字段说明**：


| 字段   | 说明           |
| ---- | ------------ |
| 报价单号 | 如 QTxxxxxxxx |
| 金额   | 报价合计         |
| 模板名  | 报价单模板名称      |


**交互**：点选一行 → 载入该报价单明细 → **评审表单**。

---

### 1.5.4 订单选单卡（按订单 · 交期路径）

**查询逻辑**：当前客户 **未排程** 订单（非「待提交」、非「已完成」；演示层见 `DemoData.isOrderUnscheduled`）；按下单日期倒序。

**字段说明**：


| 字段   | 说明                   |
| ---- | -------------------- |
| 订单号  | 如 SOxxxxxxxx         |
| 订单类型 | 五类状态徽章（见 **§1.8.0**） |
| 摘要   | 下单日期 · 金额            |


**交互**：点选一行 → 载入该订单行项目 → **评审表单**（明细只读，来自订单行）。

> **说明**：本卡 spec-id 为 `card-delivery-order-pick`，样式对齐复制/变更用 `card-order-pick`，但 data-action 为 `delivery-order-pick`。

### 1.5.5 自选商品（交期路径）

#### 1.5.5.1 需求引导卡片

**spec-id**：`card-delivery-demand`

**规则**：与 **§1.2.3.1** 一致（新客户必填需求；老客户可跳过）。

**交互**：


| 操作       | 结果              |
| -------- | --------------- |
| 确认需求     | 进入 **交期自选·选品卡** |
| 跳过（老客户）  | 按最近订单推荐 → 选品卡   |
| 对话发送完整需求 | 可跳过本卡直达选品       |


#### 1.5.5.2 交期自选·选品卡

**复用** `card-order-pick`，标题为「交期评审 · 自选商品」。

**推荐 / 筛选**：规则同 **§1.2.3.2** / **§1.4.5**。

**差异**：

- 底部主按钮文案：**下一步：确认选品**（非「逐项报价」）
- data-action：`delivery-lines-to-cart` → 汇总明细后进入 **评审表单**

### 1.5.6 评审表单

**卡片**：交期评审 · 表单（标注 `sheet-delivery`）

**出现时机**：三来源之一选定且明细就绪后，以对话内嵌表单卡片展示。

**卡片结构（自上而下）**：


| 区域   | 说明                     |
| ---- | ---------------------- |
| 卡片标题 | 「交期评审」                 |
| 来源摘要 | 只读一行 meta，随来源类型变化（见下表） |
| 表单区  | 期望交期、工艺版本明细表、是否生成采购计划  |
| 操作区  | 主按钮「提交评审」              |


**来源摘要（只读）**：


| 来源   | 展示示例             |
| ---- | ---------------- |
| 按报价单 | `报价单 QT… · ¥金额`  |
| 按订单  | `订单 SO… · 订单状态`  |
| 自选商品 | `N 项 · 品名1、品名2…` |


**表单字段**（仅以下三项由用户填写或确认；**不含**开始时间、结束时间）：


| 字段        | 类型        | 必填   | 说明                                                        |
| --------- | --------- | ---- | --------------------------------------------------------- |
| 期望交期      | 日期        | ✅    | 默认为**空**，须用户选择；未填 toast「请选择期望交期」                          |
| 工艺版本（按货品） | 明细表每行下拉   | ✅ 每行 | 表头 `# / 品名 / 工艺版本`；候选项按货品规格从主数据读取；任一行未选 toast「请为每项选择工艺版本」 |
| 是否生成采购计划  | 单选（是 / 否） | ✅    | **默认值由后台带入（已实现）**；表单以单选展示，**用户可修改**后再提交                   |


**交互**：


| 操作   | 结果                                                                                              |
| ---- | ----------------------------------------------------------------------------------------------- |
| 提交评审 | 校验期望交期 + 每行工艺版本 → 连同用户确认后的「是否生成采购计划」调用后台评审 → 关闭浮层 → 写入会话交期评审结果（`confirmed: true`）→ 推送 **评审结果卡** |


**写入会话（字段摘要）**：


| 字段        | 说明                           |
| --------- | ---------------------------- |
| 来源类型      | 按报价单 / 按订单 / 自选商品            |
| 来源标识      | 报价单号、订单号或自选行明细               |
| 期望交期      | 用户填写                         |
| 工艺版本（按货品） | 用户填写（按行）                     |
| 是否生成采购计划  | 默认后台带入（已实现）；以用户提交时的选择为准      |
| 开始时间      | 后台返回（已实现）                    |
| 结束时间      | 后台返回（已实现）                    |
| 评审结果      | 后台返回（已实现）；含结论徽章、按期时的结论文案与说明行 |
| 无法按时交付原因  | 后台返回（已实现）；仅评审结果为「无法按时交付」时有值  |
| 已确认       | 提交后为真                        |

### 1.5.7 评审结果卡

**卡片**：交期评审 · 结果（标注 `card-delivery`）

**出现时机**：评审表单「提交评审」校验通过且后台返回后，以对话气泡内卡片展示。

**卡片结构（自上而下）**：


| 区域   | 说明                              |
| ---- | ------------------------------- |
| 卡片标题 | 「交期评审 ·」+ **结论徽章**（后台返回，已实现）    |
| 结论区  | **仅评审结果为「按期」时展示**（后台返回，已实现）     |
| 原因区  | **仅评审结果为「无法按时交付」时展示**（后台返回，已实现） |
| 参数摘要 | 浅灰底标签-值区（见下表）；**均为只读展示，不可修改**   |
| 操作区  | 主按钮 1 个；无法按时交付时附加「调整方案」         |


**结论徽章**（标题右侧，后台返回，已实现）：


| 评审结果   | 徽章文案   | 样式                   |
| ------ | ------ | -------------------- |
| 按期     | 按期     | 成功绿（`sc-badge--new`） |
| 无法按时交付 | 无法按时交付 | 警告橙（`sc-badge--old`） |


**结论区**（后台返回，已实现；**仅「按期」时展示**）：


| 元素   | 说明                    |
| ---- | --------------------- |
| 结论文案 | 绿色强调，如「可以按时交付」        |
| 说明行  | 灰色 meta，如「预计 … 可整单交付」 |


> **无法按时交付**时不展示结论区：无橙色结论文案、无说明行、无「无法按时交付原因」标题。

**原因区**（后台返回，已实现；**仅「无法按时交付」时展示**）：


| 元素   | 说明                                             |
| ---- | ---------------------------------------------- |
| 展示形式 | 浅黄底原因框（`sc-delivery-result__blockers`），**无标题** |
| 内容   | 后台返回的无法按时交付原因列表；按明细行逐条展示                       |


**参数摘要**（`sc-delivery-result__summary`，标签-值对齐，浅灰底）：


| 标签          | 说明                                      |
| ----------- | --------------------------------------- |
| 开始时间 / 结束时间 | 后台返回（已实现）；**同一行两列**展示，日期格式 `YYYY/MM/DD` |
| 期望交期        | 表单填写回显                                  |
| 工艺版本        | 按货品**分行**列表：品名（加粗）+ 工艺版本号               |
| 采购计划        | **只读展示**（是 / 否）；取评审表单提交时的选择；**结果卡不可修改** |


**操作区（按来源仅展示一个主按钮）**：


| 来源          | 主按钮        | 说明                  |
| ----------- | ---------- | ------------------- |
| 按报价单 / 自选商品 | **生成订单**   | 预填待确认订单 → **下单确认卡** |
| 按订单         | **查看订单进度** | 直达所选订单的 **进度详情卡**   |


**无法按时交付时附加**：线框按钮「**调整方案**」→ 进入方案速配入口。

> 结果卡**不展示**「评审完成，可以生成订单。」等引导文案。

**与主链路关系**：交期评审为**独立可选能力**；报价单卡、逐项报价、复制订单等「生成订单」**不**要求先完成交期。结果卡「生成订单」仅为交期完成后的便捷入口；若本会话已评审，下单确认卡可展示交期摘要行。

## 1.6 复制订单

### 1.6.1 选单卡

**前置条件**：顶栏已选客户；本客户至少有 1 笔历史订单。

**内容**：与变更共用 `card-order-pick`，列表展示订单号、**订单类型**徽章、日期 · 金额。

**交互**：


| 操作              | 结果                    |
| --------------- | --------------------- |
| 点选一行（copy-pick） | 载入该订单明细 → **复制明细确认卡** |


**无单时**：对话提示「该客户暂无历史订单可复制」，不推选单卡。

---

### 1.6.2 复制明细确认卡

**spec-id**：`card-order-copy`

**出现时机**：选单复制后；**不可**复用 **§1.4.7 下单确认卡**（本卡侧重明细核对与行级编辑）。

**内容**：


| 块    | 说明                  |
| ---- | ------------------- |
| 标题   | 复制订单 · 明细确认         |
| 来源订单 | 订单号、订单类型徽章、下单日期     |
| 客户   | 当前客户名称              |
| 提示   | 点击明细行展开，可修改规格、数量与单价 |
| 明细列表 | 可点击行（见下）            |
| 合计金额 | 随明细编辑实时更新           |


**明细行（可点击）**：


| 状态          | 展示                                       |
| ----------- | ---------------------------------------- |
| 收起（默认仅首行展开） | 序号、品名、数量 · 小计 · 规格摘要、展开箭头                |
| 展开          | 规格 select、数量、工艺版本、税率、单价；行小计；低于最低售价时行边框警告 |


**字段编辑规则**：与 **§1.3.3.6 逐项报价卡** 行级字段一致（规格变更联动最新/最低售价；改价标记 `_quotePriceTouched`）。

**交互**：


| 操作       | 结果                                                 |
| -------- | -------------------------------------------------- |
| 点击明细行    | 展开/收起该行编辑区（手风琴，同时仅一行展开）                            |
| 修改行内字段   | 实时重算行小计与合计                                         |
| **确认下单** | 同步 DOM → 进入 **§1.4.7 下单确认卡**（订单来源展示「复制订单 · SOxxx」） |


**与下单确认卡的分工**：


| 卡片                | 职责                                  |
| ----------------- | ----------------------------------- |
| `card-order-copy` | 明细可点选、可改价改量                         |
| `sheet-order`     | 结算方式/货币、发货日期、交期评审摘要（如有）；明细只读表格；最终提交 |


**业务规则**：

- `sourceType: 'copy'`，写入 `copiedOrderId`
- 提交后新单默认状态 **未审核**（见 **§1.4.8** 增量）

## 1.8 订单进度

### 1.8.0 订单类型（全产品统一）

v1.3.0 起列表、选单、进度详情统一使用以下五类（替代 v1.2.0 的待排产/生产中/已发货等文案）：


| 类型       | 含义（演示）        | 列表徽章 |
| -------- | ------------- | ---- |
| **未审核**  | 新建/刚提交，待内勤处理  | 中性灰  |
| **销售审核** | 销售主管审核中       | 主色描边 |
| **已审核**  | 审核通过，可排产/履约   | 成功绿  |
| **已完成**  | 整单关闭          | 弱化灰  |
| **异常**   | 缺料、交期冲突、变更驳回等 | 警告色  |


另：**待提交** 为报价转单的草稿态，不在进度列表与交期「按订单」选单中展示。

---

### 1.8.1 进度列表卡

**spec-id**：`card-order-progress-list`

**查询逻辑**：当前客户全部订单，按下单日期倒序。

**字段说明**：


| 字段   | 说明                  |
| ---- | ------------------- |
| 订单号  | SOxxxxxxxx          |
| 订单类型 | 五类徽章                |
| 状态说明 | `statusDetail` 一行摘要 |
| 品项摘要 | 品名×数量 · 日期          |
| 查看详情 | 链接按钮                |


**交互**：点「查看详情」→ **进度详情卡**。

---

### 1.8.2 进度详情卡

**spec-id**：`card-order-progress-detail`

**内容**：


| 块          | 说明     |
| ---------- | ------ |
| 订单号 + 类型徽章 | 标题区    |
| 状态说明       | 当前节点描述 |
| 品项 · 金额    | 摘要     |
| 审核履约时间轴    | 见下     |


**时间轴规则**：

- 标准节点：**未审核 → 销售审核 → 已审核 → 已完成**
- 当前节点高亮（`current`）；已完成节点打勾（`done`）
- **异常**：对应节点标红（`error`），展示 `statusDetail`
- **已完成**：全节点打勾

**入口**：

- 技能「订单进度」
- 交期结果卡（按订单来源）「查看订单进度」
- 话术「查看订单 xxx 进度」

## 附录 A：v1.3.0 对 v1.2.0 主链路的变更


| 章节              | v1.2.0         | v1.3.0                                     |
| --------------- | -------------- | ------------------------------------------ |
| 主链路             | 方案 → 报价 → 下单   | 主链路不变；**交期评审**为并列可选能力                      |
| **§1.4.7 下单确认** | 无交期行；来源无「复制订单」 | 附加 **交期评审** 摘要行（本会话已评审时）；来源含「复制订单 · SOxxx」 |
| **§1.4.7 提交前**  | 无交期前置          | **不要求**先交期；仅展示本会话交期摘要（如有）                  |
| **§1.4.8 成功卡**  | 状态「待排产」        | 状态 **未审核**；时间轴种子写入                         |
| 订单进度            | 演示级            | 五类状态 + 时间轴详情（**§1.8**）                     |

## 附录 B：v1.3.0 增量业务规则


| 规则      | 说明                                                                |
| ------- | ----------------------------------------------------------------- |
| 客户必选    | 交期 / 复制 / 变更 / 进度均须已选客户                                           |
| 交期三来源   | 按报价单、按订单、自选商品，互斥；**按订单**须本客户有未排程订单                                |
| 评审表单三字段 | 期望交期、工艺版本（按行）、是否生成采购计划（默认后台带入、可改）；开始/结束时间、评审结果、无法按时交付原因由后台返回（已实现） |
| 交期与下单   | 并列能力；生成订单**不**以交期为先决条件                                            |
| 复制两阶段确认 | `card-order-copy` 改明细 → `sheet-order` 结算提交                        |
| 变更限制    | 已完成不可变更；已审核变更后回销售审核                                               |
| 对话失效    | 同 v1.2.0 **附录**                                                   |
