# 功能描述 · 交期评审 / 复制订单 / 订单变更 / 订单进度（v1.3.0）

> 本文档按「入口卡 → 来源/选单 → 分步卡片」结构描述 **v1.3.0 增量能力**，供产品、研发与验收对齐。  
> **基线**：[v1.2.0 功能描述 · 方案报价下单](../v1.2.0/功能描述-方案报价下单-v1.2.0.md)（方案速配 / 产品报价 / 确认下单主链路不变）  
> **对应实现**：`v1.3.0/` · 标注详见 [annotation-docs/05-交期与订单运营.md](./annotation-docs/05-交期与订单运营.md)

---

## 0. 本版范围

| 能力 | 说明 |
|------|------|
| **交期评审** | 三来源（按报价单 / 按订单 / 自选商品）→ 评审表单 → 评审结果；与下单**并列**，不阻塞报价/下单主链路 |
| **复制订单** | 选历史单 → 明细确认（可改价改量）→ 下单确认 → 提交 |
| **订单变更** | 选单 → 变更原因 + 备注 → 提交成功 |
| **订单进度** | 五类订单类型 + 列表 + 详情时间轴 |

**全局前置**：交期 / 复制 / 变更 / 进度均须顶栏已选当前客户。

**主流程关系**：

```
v1.2.0：方案速配 → 产品报价 → 确认下单（不变）

v1.3.0 增量（均可从技能条或话术进入，互不强制串联）：
  交期评审 ──可选──→ 结果卡「生成订单」→ 下单确认
  复制订单 ──────────→ 明细确认 → 下单确认
  订单变更 / 订单进度 ──独立查询与办理
```

## 1.5 交期评审

### 1.5.1 入口卡

**前置条件**：顶栏已选当前客户。

**出现时机**：技能「交期评审」、欢迎区快捷入口、话术「评估交期」「什么时候能交货」等；技能条可**直达来源选择**，不经本卡。

**内容**：

- 标题：交期评审
- 当前客户名称、客户类型标签（新客户 / 老客户）
- 主按钮：**评估交期**

**字段说明**：

| 字段 | 说明 |
|------|------|
| 客户名称 | 顶栏选中客户 |
| 客户类型 | 规则同 v1.2.0 **§1.2.1**（新/老客户判定） |

**交互**：

| 操作 | 结果 |
|------|------|
| 评估交期 | 进入 **交期评审 · 选择来源** |

---

### 1.5.2 选择来源卡

**标题**：交期评审 · 选择来源

**内容**（三选一，互斥）：

| 选项 | 说明 | 可用条件 |
|------|------|----------|
| **按报价单** | 以报价单明细评估交期 | 本客户至少有 1 份报价单；否则置灰 |
| **按订单** | 以未排程订单明细评估交期 | 本客户至少有 1 笔**未排程**订单；否则置灰 |
| **自选商品** | 直选品 + 规格 + 数量后评估 | 始终可用 |

**副文案**：展示「共 N 份历史报价单」或「共 N 笔未排程订单」；仅 1 份/1 笔时提示将直接进入评审表单。

**查询逻辑**：

- 报价单：当前客户全部报价单
- 未排程订单：非「待提交」、非「已完成」的在途订单（演示层按订单状态过滤）

**交互**：

| 操作 | 结果 |
|------|------|
| 按报价单 | 仅 **1** 份，或会话已有当前报价单 → **评审表单**；多份 → **选报价单** |
| 按订单 | 仅 **1** 笔未排程 → **评审表单**；多笔 → **选订单（交期）** |
| 自选商品 | 新客户先 **需求引导**（规则同 **§1.2.3.1**）→ **交期自选·选品** → **下一步：确认选品** → **评审表单** |

---

### 1.5.3 选报价单卡

**出现时机**：按报价单且本客户报价单 ≥ 2。

**查询逻辑**：当前客户全部报价单，按生成时间倒序。

**字段说明**：

| 字段 | 说明 |
|------|------|
| 报价单号 | 如 QTxxxxxxxx |
| 金额 | 报价合计 |
| 模板名 | 报价单模板名称 |

**交互**：点选一行 → 载入报价明细（只读）→ **评审表单**。

---

### 1.5.4 选订单卡（交期路径）

**出现时机**：按订单且本客户未排程订单 ≥ 2。

**查询逻辑**：当前客户未排程订单，按下单日期倒序。

**字段说明**：

| 字段 | 说明 |
|------|------|
| 订单号 | 如 SOxxxxxxxx |
| 订单类型 | 五类状态徽章（见 **§1.8.0**） |
| 摘要 | 下单日期 · 金额 |

**交互**：点选一行 → 载入订单行项目（只读）→ **评审表单**。

> 列表样式与复制/变更用 **订单选单卡** 相近，交期场景下仅点选带入明细，勿与复制/变更点选混用。

---

### 1.5.5 自选商品

#### 1.5.5.1 需求引导卡

**出现时机**：自选商品路径，且须先采集采购需求时（规则同 **§1.2.3.1**）。

**内容**：

- 采购需求多行输入 +「确认需求」
- 老客户：**跳过，按最近订单推荐**
- 对话一句说齐需求时可**跳过本卡**直达选品

**交互**：

| 操作 | 结果 |
|------|------|
| 确认需求 | **交期评审 · 自选商品**选品卡 |
| 跳过（老客户） | 按最近订单推荐进入选品卡 |

#### 1.5.5.2 交期自选·选品卡

**标题**：交期评审 · 自选商品（复用 **订单选单卡** 宿主，交期模式）

**推荐 / 筛选**：规则同 **§1.2.3.2**、报价选品 **§1.3.3.4**。

**与报价选品差异**：

| 项 | 交期自选 | 报价选品 |
|----|----------|----------|
| 底部主按钮 | **下一步：确认选品** | 逐项报价 / 进购物车 |
| 下一步结果 | 汇总明细 → **评审表单** | 报价选品确认 / 逐项报价 |

---

### 1.5.6 评审表单卡

**出现时机**：三来源之一明细就绪后，以对话内嵌表单展示。

**卡片结构（自上而下）**：

| 区域 | 说明 |
|------|------|
| 标题 | 交期评审 |
| 来源摘要 | 只读一行，随来源变化（见下表） |
| 表单区 | 期望交期、工艺版本明细表、是否生成采购计划 |
| 操作区 | **提交评审** |

**来源摘要（只读）**：

| 来源 | 展示示例 |
|------|----------|
| 按报价单 | 报价单 QT… · ¥金额 |
| 按订单 | 订单 SO… · 订单状态 |
| 自选商品 | N 项 · 品名1、品名2… |

**表单字段**（用户填写/确认；**不含**开始时间、结束时间）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 期望交期 | 日期 | ✅ | 默认为**空**；未选 toast「请选择期望交期」 |
| 工艺版本（按货品） | 明细表每行下拉 | ✅ 每行 | 表头：序号 / 品名 / 工艺版本；候选项按货品+规格从主数据读取；任一行未选 toast「请为每项选择工艺版本」 |
| 是否生成采购计划 | 单选（是 / 否） | ✅ | **默认值由后台带入**；用户可修改后再提交 |

**提交后写入会话（摘要）**：

| 字段 | 说明 |
|------|------|
| 来源类型 | 按报价单 / 按订单 / 自选商品 |
| 来源标识 | 报价单号、订单号或自选行明细 |
| 期望交期、工艺版本、是否生成采购计划 | 以用户提交为准 |
| 开始时间、结束时间 | 演示层提交时生成计划区间（默认今日起 +7 / +14 天），结果卡展示 |
| 评审结果 | 按期 / 无法按时交付；结论文案、说明行或原因列表由评审逻辑返回 |
| 已确认 | 提交成功后为真 |

**交互**：

| 操作 | 结果 |
|------|------|
| 提交评审 | 校验期望交期 + 每行工艺版本 → 写入会话评审结果 → 推送 **评审结果卡** |

**齐套演示规则（原型）**：期望交期早于计划结束日，且采购计划为「是」时倾向「按期」；否则「无法按时交付」并列出原因行。

---

### 1.5.7 评审结果卡

**出现时机**：评审表单提交通过并生成评审结果后。

**卡片结构（自上而下）**：

| 区域 | 说明 |
|------|------|
| 标题 | 交期评审 · + **结论徽章**（按期 / 无法按时交付） |
| 结论区 | **仅「按期」**：绿色结论文案 + 灰色说明行（如「预计 … 可整单交付」） |
| 原因区 | **仅「无法按时交付」**：浅黄底原因列表，**无标题** |
| 参数摘要 | 浅灰底只读区（见下表） |
| 操作区 | 主按钮 1 个；无法按时交付时附加「调整方案」 |

**参数摘要（只读，不可改）**：

| 标签 | 说明 |
|------|------|
| 开始时间 / 结束时间 | 同一行两列；日期格式 YYYY/MM/DD |
| 期望交期 | 表单填写回显 |
| 工艺版本 | 按货品分行：品名 + 版本号 |
| 采购计划 | 是 / 否；取表单提交时的选择 |

**操作区（按来源仅 1 个主按钮）**：

| 来源 | 主按钮 | 结果 |
|------|--------|------|
| 按报价单 / 自选商品 | **生成订单** | 预填待确认订单 → **下单确认卡**（v1.2.0 **§1.4.7**） |
| 按订单 | **查看订单进度** | **订单进度 · 详情**（所选订单） |

**无法按时交付时**：线框按钮「**调整方案**」→ 进入方案速配。

> 不展示「评审完成，可以生成订单。」等冗余引导文案。

**与主链路关系**：报价单卡、复制订单等路径的「生成订单」**不要求**先完成交期；若本会话已评审，**下单确认卡**可展示交期摘要行（只读）。

## 1.6 复制订单

### 1.6.1 选单卡

**前置条件**：顶栏已选客户；本客户至少有 1 笔历史订单。

**内容**：复用 **订单选单卡**（复制模式）；列表展示订单号、五类订单类型徽章、下单日期 · 金额。

**查询逻辑**：当前客户全部历史订单，按下单日期倒序。

**交互**：

| 操作 | 结果 |
|------|------|
| 点选一行 | **复制订单 · 明细确认** |
| 无历史订单 | 对话提示，不推选单卡 |

---

### 1.6.2 复制明细确认卡

**出现时机**：复制选单之后；本步负责明细核对与行级改价，**不**替代下单确认。

**内容**：

| 块 | 说明 |
|----|------|
| 标题 | 复制订单 · 明细确认 |
| 来源 | 订单号、订单类型徽章、下单日期 |
| 客户 | 当前客户名称 |
| 提示 | 点击明细行可改规格、数量、单价 |
| 明细列表 | 手风琴行（见下） |
| 合计金额 | 随编辑实时更新 |

**明细行**：

| 状态 | 展示 |
|------|------|
| 收起 | 序号、品名、数量 · 小计、规格摘要 |
| 展开 | 规格、数量、工艺版本、税率、单价；低于最低售价时行边框警告 |

**字段编辑规则**：与 v1.2.0 **§1.3.3.6 逐项报价卡** 行级规则一致（规格变更联动售价；手动改价标记已改价）。

**交互**：

| 操作 | 结果 |
|------|------|
| 点击明细行 | 展开/收起（同时仅一行展开） |
| 修改行内字段 | 重算行小计与合计 |
| **确认下单** | 进入 **下单确认卡**；订单来源展示「复制订单 · SOxxx」 |

**两卡分工**：

| 卡片 | 职责 |
|------|------|
| 复制明细确认 | 明细可点选、可改价改量 |
| 下单确认 | 结算方式/货币、发货日期、交期摘要（如有）；明细只读表格；最终提交 |

**业务规则**：

- 来源记为「复制订单」，保留被复制订单号
- 提交后新单状态为 **未审核**（见 **§1.8.0**、附录 A）

## 1.7 订单变更

### 1.7.1 选单卡

**前置条件**：顶栏已选客户；本客户至少有 1 笔订单。

**内容**：复用 **订单选单卡**（变更模式）；字段同 **§1.6.1**。

**交互**：

| 操作 | 结果 |
|------|------|
| 点选一行 | **订单变更 · 表单** |

**限制**：

| 订单类型 | 行为 |
|----------|------|
| 已完成 | Toast「已完成订单不可变更」，不打开表单 |
| 其他 | 允许发起变更 |

---

### 1.7.2 变更表单卡

**标题**：变更订单 — {订单号}

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 变更原因 | 下拉 | 建议填 | 候选项见演示主数据 |
| 变更备注 | 多行文本 | ✅ | 为空 toast「请填写变更备注」 |

**交互**：

| 操作 | 结果 |
|------|------|
| 提交变更 | 已审核订单 → 状态回退为 **销售审核**，状态说明更新；其他状态更新说明文案 → **变更已提交**卡 |

---

### 1.7.3 变更已提交卡

**内容**：订单号、变更原因、变更备注（只读确认）。

**交互**：停留对话页展示，无强制后续跳转。

## 1.8 订单进度

### 1.8.0 订单类型（全产品统一）

v1.3.0 起列表、选单、进度详情统一使用以下五类（替代 v1.2.0「待排产 / 生产中 / 已发货」等文案）：

| 类型 | 含义（演示） | 列表徽章 |
|------|--------------|----------|
| **未审核** | 新建/刚提交，待内勤处理 | 中性灰 |
| **销售审核** | 销售主管审核中 | 主色描边 |
| **已审核** | 审核通过，可排产/履约 | 成功绿 |
| **已完成** | 整单关闭 | 弱化灰 |
| **异常** | 缺料、交期冲突、变更驳回等 | 警告色 |

**待提交**：报价转单草稿，不在进度列表与交期「按订单」选单中展示。

---

### 1.8.1 进度列表卡

**查询逻辑**：当前客户全部订单（含五类状态），按下单日期倒序。

**字段说明**：

| 字段 | 说明 |
|------|------|
| 订单号 | SOxxxxxxxx |
| 订单类型 | 五类徽章 |
| 状态说明 | 一行摘要文案 |
| 品项摘要 | 品名×数量 · 日期 |
| 查看详情 | 链接按钮 |

**交互**：点「查看详情」→ **订单进度 · 详情**。

---

### 1.8.2 进度详情卡

**内容**：

| 块 | 说明 |
|----|------|
| 标题 | 订单号 + 类型徽章 |
| 状态说明 | 当前节点描述 |
| 品项 · 金额 | 摘要 |
| 审核履约时间轴 | 见下 |

**时间轴规则**：

- 标准节点：**未审核 → 销售审核 → 已审核 → 已完成**
- 当前节点高亮；已完成节点打勾
- **异常**：对应节点标红，展示状态说明
- **已完成**：全节点打勾

**入口**：

- 技能「订单进度」
- 交期结果卡（按订单来源）「查看订单进度」
- 话术「查订单进度」「订单到哪了」等

**交互**：只读详情，无本卡内提交操作。

## 附录 A：v1.3.0 对 v1.2.0 主链路的变更

| 章节 | v1.2.0 | v1.3.0 |
|------|--------|--------|
| 主链路 | 方案 → 报价 → 下单 | **不变**；交期评审为**并列可选**能力 |
| **§1.4.7 下单确认** | 无交期行；来源无「复制订单」 | 可展示**交期评审摘要**（本会话已评审）；来源含「复制订单 · SOxxx」 |
| 提交前 | 无交期前置 | **不要求**先交期；仅展示已有交期摘要 |
| **§1.4.8 订单成功** | 状态「待排产」 | 新单默认 **未审核**；可种子写入进度时间轴 |
| 订单进度 | 演示级 | 五类状态 + 列表 + 时间轴详情（**§1.8**） |

## 附录 B：业务规则汇总

| 规则 | 说明 |
|------|------|
| 客户必选 | 交期 / 复制 / 变更 / 进度均须已选客户 |
| 交期三来源 | 按报价单、按订单、自选商品互斥；按订单须有未排程订单 |
| 评审表单 | 用户填：期望交期、工艺版本（按行）、是否生成采购计划；开始/结束时间在结果卡展示 |
| 交期与下单 | **并列**；生成订单不以交期为先决条件 |
| 复制两阶段 | 明细确认改价 → 下单确认结算提交 |
| 变更限制 | 已完成不可变更；已审核变更后回 **销售审核** |
| 对话失效 | 同 v1.2.0：新卡出现后旧卡按钮失效 |
| 话术示例 | 评估交期、按报价单评估交期、复制老订单、变更订单、查进度等 |
