## 二、交期评审（§1.5）

### 2.1 入口卡 `card-delivery-entry`

**内容**：功能名「交期评审」、当前客户名称、客户类型标签（新/老客户）；主操作「评估交期」。

**查询**：顶栏已选客户；客户类型规则同方案速配 §1.2.1。

**交互**：`delivery-create-new` → **来源选择卡**（技能条直达时经 `runDelivery` 可跳过本卡，直接来源卡）。

### 2.2 来源选择卡 `card-delivery-source`

**内容**（三选一互斥）：

| 选项 | 可用条件 | 副文案 |
|------|----------|--------|
| 按报价单 | ≥1 份报价单 | 共 N 份历史报价单 / 暂无历史报价单 |
| 按订单 | ≥1 笔**未排程**订单 | 共 N 笔未排程订单 / 暂无未排程订单 |
| 自选商品 | 始终可用 | 直选品、规格与数量评审交期 |

**查询**：

- 报价单：`quotesForCustomer`
- 未排程订单：`deliveryOrdersForCustomer` / `DemoData.isOrderUnscheduled`（非待提交、非已完成）

**交互**：

| data-action | 结果 |
|-------------|------|
| `delivery-source-quote` | 1 份或会话当前报价单 → `sheet-delivery`；多份 → `card-delivery-quote-pick` |
| `delivery-source-order` | 1 笔未排程 → `sheet-delivery`；多笔 → `card-delivery-order-pick` |
| `delivery-source-lines` | 新客户需求引导 → 交期自选选品 → `sheet-delivery` |

### 2.3 报价单选单 `card-delivery-quote-pick`

**查询**：当前客户全部报价单，按生成时间倒序。

**内容**：报价单号、金额、模板名。

**交互**：`delivery-quote-pick` + `data-quote-id` → 载入明细 → `sheet-delivery`。

### 2.4 订单选单（交期）`card-delivery-order-pick`

**查询**：当前客户**未排程**订单，按下单日期倒序。

**内容**：订单号、五类状态徽章、日期 · 金额。

**交互**：`delivery-order-pick` + `data-oid` → 只读订单行 → `sheet-delivery`。

> 与复制/变更共用样式的 `card-order-pick` 不同 action，勿混用。

### 2.5 需求引导 `card-delivery-demand`

**内容**：采购需求输入；老客户可「跳过，按最近订单推荐」。

**交互**：规则同 §1.2.3.1；`delivery-demand-submit` / `delivery-skip-demand` → 交期自选·选品卡。

### 2.6 交期自选·选品 `card-order-pick`（交期模式）

**识别**：标题「交期评审 · 自选商品」；`ctx.deliveryLinesMode === true`。

**查询 / 推荐**：同 §1.2.3.2 / §1.4.5 选品规则。

**交互**：主按钮「下一步：确认选品」`delivery-lines-confirm` → 汇总明细 → `sheet-delivery`。

### 2.7 评审表单 `sheet-delivery`

**内容（只读）**：来源摘要（报价单号/订单号/品项数）。

**表单字段**：

| 字段 | data-field / action | 必填 |
|------|---------------------|------|
| 期望交期 | `delivery-expected-date` | ✅ |
| 工艺版本（按行） | `delivery-line-process` + `data-idx` | ✅ 每行 |
| 是否生成采购计划 | `delivery-procurement` radio | ✅ |
| 开始时间 | `delivery-plan-start` | ✅ |
| 结束时间 | `delivery-plan-end` | ✅，≥ 开始 |

**交互**：`delivery-submit` → 写入 `ctx.delivery`（`confirmed: true`）→ `card-delivery`。

**查询（齐套演示）**：期望交期 ≥ 计划结束且采购计划=是 → 倾向按期；否则不齐套 + blockers。

### 2.8 评审结果 `card-delivery`

**内容**：结论徽章（按期/不齐套）、结论文案、期望交期与计划区间、工艺版本摘要、采购计划、不齐套原因列表。

**交互（按来源单一主按钮）**：

| 来源 | 主按钮 | data-action |
|------|--------|-------------|
| quote / lines | 生成订单 | `delivery-to-order` → 下单确认卡 |
| order | 查看订单进度 | `delivery-to-progress` + `data-oid` |

不齐套附加：`skill-plan`（调整方案）。

## 三、复制订单（§1.6）

### 3.1 选单 `card-order-pick`（复制模式）

**前置**：已选客户；≥1 笔历史订单。

**交互**：`copy-pick` → `card-order-copy`；无单时文案提示，不出卡。

### 3.2 明细确认 `card-order-copy`

**内容**：来源订单号/状态/日期、客户、可点击明细行、合计金额。

**交互**：`copy-line-toggle` 手风琴展开编辑；`copy-order-confirm` → `sheet-order`（来源「复制订单 · SOxxx」）→ 提交后 **未审核**。

---

## 四、订单变更（§1.7）

### 4.1 选单 `card-order-pick`（变更模式）

**交互**：`change-pick` → `sheet-change`；**已完成** Toast 拦截。

### 4.2 变更表单 `sheet-change`

**字段**：变更原因 select（`change-reason`）、备注 textarea（`change-remark`，必填）。

**交互**：`change-submit` → 已审核回退销售审核 → `card-change-success`。

### 4.3 变更成功 `card-change-success`

**内容**：订单号、原因、备注只读确认。

## 五、订单进度（§1.8）

### 5.1 五类订单状态

| status | 徽章 | 说明 |
|--------|------|------|
| 未审核 | muted | 待内勤 |
| 销售审核 | primary | 主管审核中 |
| 已审核 | new | 可排产/履约 |
| 已完成 | done | 整单关闭 |
| 异常 | old | 需处理 |

**待提交**：报价转单草稿，不在进度列表与交期按订单选单中。

### 5.2 进度列表 `card-order-progress-list`

**查询**：当前客户全部订单，下单日期倒序。

**内容**：订单号、类型徽章、statusDetail、品项 · 日期、「查看详情」。

**交互**：`progress-detail` → `card-order-progress-detail`。

### 5.3 进度详情 `card-order-progress-detail`

**内容**：订单号 + 徽章、状态说明、品项 · 金额、审核履约时间轴。

**时间轴**：未审核 → 销售审核 → 已审核 → 已完成；异常节点标红。
