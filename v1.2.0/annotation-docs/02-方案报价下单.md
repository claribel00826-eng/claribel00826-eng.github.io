# 标注文档 02：方案速配 · 产品报价 · 确认下单

> **版本**：1.2.0 版 · **入口**：欢迎区/技能条/意图话术  
> **前置**：多数步骤须已选**当前客户**（顶栏展示）；缺客户时出选客户引导卡。  
> **上下文**：会话内按客户保存方案、报价单、待确认报价、待确认订单等（按客户校验）。

---

## 一、主链路与解耦

```
选客户
  → 方案速配：选品改规格 → 购物车改规格 → 方案模板 → 方案卡（无价格）
  → 产品报价：按方案 / 直选品 → 逐项报价 → 报价单模板 → 报价单卡
  → 确认下单：按报价单 / 直选品（须逐项报价）→ 下单确认抽屉 → 待排产成功卡
```

| 原则 | 说明 |
|------|------|
| 方案无价格 | 方案卡、选品列表**不展示单价**；金额仅在报价阶段 |
| 报价≠下单 | 生成报价单**不**自动建单；须报价单卡「生成订单」或技能「确认下单」 |
| 一句话捷径 | 描述性话术可跳过中间卡片（详见文档 **03** §三、§四） |
| 历史按钮 | 用户新发消息后，上一步助手卡按钮失效（文档 **03** 第二节） |

## 二、方案速配

### 2.1 入口与引导

| 入口 | 行为 |
|------|------|
| 欢迎区「方案速配」/ 技能条 | 发话术「配个方案」→ 引导句 + **选品卡** |
| 待跟进「做方案」 | 切换客户后进入方案速配 |
| 一句话需求 | 直接 **方案卡**（03 §三） |

**引导句内容**：说明可一句话描述需求，或走选品 → 购物车 → 模板；强调方案不含价格。

### 2.2 方案选品卡

#### 内容

1. 标题「选品 · 改规格」。
2. **推荐区**（列表上部，按客户类型分路）  
   - **老客户**：本客户历史订单中的 productIds，按订单日期倒序，去重后最多 10 条；标签「历史订单」；**不使用**存货描述匹配。  
   - **新客户**：客户档案 `demandHint`（需求关键词）与每条存货的 **`inventoryDesc`（存货描述字段）** 做分词命中计分（品名不参与推荐区计分）；得分 **>0.8** 取前 10 条，按分降序；标签「匹配 xx%」。  
3. **筛选**：搜索框 +「筛选」按钮；对**推荐区与更多产品**同时过滤品名 / 规格 / 存货描述。
4. **更多产品**（推荐区未展示的关联存货，见下「查询逻辑」）  
   - 标签「全部」；**首屏 5 条**，列表内下滑懒加载每页 5 条（03 §五）。  
5. 每行：品名、规格、匹配标签；**规格下拉始终可见**（未勾选也可改规格）。
6. 底部：「加入购物车」；语音提示文案。

#### 查询逻辑

**关联查询总则**

| 客户类型 | 需求文本来源 | 匹配字段（推荐/更多） |
|----------|--------------|------------------------|
| 老客户 | 无（走订单） | 推荐区：订单 productIds；更多区：筛选词 → 品名/规格/存货描述 |
| 新客户 | 客户 `demandHint` | 推荐区与更多区均用 **`inventoryDesc`（存货描述）** 与 demandHint 计分 |

**计分规则（仅新客户、仅存货描述字段）**  
将需求文本切分为长度 ≥2 的关键词，统计在 `inventoryDesc` 中命中的占比，得 0～1 分。

**推荐区**

- 老客户：历史订单倒序 Top10。  
- 新客户：分 **>0.8** → Top10。

**更多产品**

- 候选集 = 演示产品全库 **减去** 推荐区已展示品项。  
- **老客户**：通过筛选词过滤（三字段包含）；无筛选词则展示全部余量；按品名升序。  
- **新客户**：`demandHint` × `inventoryDesc` 得分 ∈ **(0.2, 0.8]**（弱关联，推荐区已取 >0.8）；或用户输入筛选词且品名/规格/存货描述任一命中；按得分降序。  
- 分页：`moreVisible` 初值 5，筛选后重置为 5，触底每次 +5。

**上下文**

- 方案流程：已选品、规格、数量、筛选词、更多区可见条数。  
- 默认规格来自演示主数据。

#### 交互逻辑

| 操作 | 结果 |
|------|------|
| 点行勾选 | 勾选/取消已选品 |
| 改规格下拉 | 更新该行规格 |
| 筛选 / 语音「筛选 xxx」 | 更新筛选词，重置更多区分页，刷新卡片 |
| 语音「选品 伺服电机」 | 按关键词勾选并刷新 |
| 加入购物车 | 至少 1 项 → 出 **方案购物车卡** |
| 语音「加购」 | 同左 |

### 2.3 方案购物车卡

#### 内容

- 已选行：品名、规格下拉、数量输入。
- 「返回选品」「生成方案」；提示须选模板。

#### 交互

- 改数量与规格。  
- **生成方案** → 打开 **方案模板抽屉**（非直接出方案卡）。  
- 语音「生成方案」→ 打开模板抽屉。

### 2.4 方案模板抽屉

#### 内容

- 单选一种方案模板（名称 + 描述）。

#### 交互

- 未选模板时轻提示。  
- 确认后写入当前客户方案 → 关闭抽屉 → 会话插入 **方案卡**。  
- 方案行：品名·规格·数量；**无价格字段展示**。

### 2.5 方案卡

#### 内容

- 方案号、客户、模板名、行明细（品名 · 规格 · 数量）。  
- 按钮：「预览文件」「去报价」。

#### 查询

- 当前客户方案：编号、客户、模板、行明细

#### 交互

- **去报价**：无方案时提示先完成方案速配；有方案 → 载入方案行 → **逐项报价抽屉**（或一句话直接报价单，见文档 03 第四节）。  
- 无「生成订单」按钮（须先报价）。

## 三、产品报价

### 3.1 报价来源卡

#### 内容

- 「按方案报价」「直接选品报价」；有方案时展示方案号提示。

#### 查询

- 本客户是否已有方案。

#### 交互

| 按钮 | 行为 |
|------|------|
| 按方案 | 方案行 → 待确认报价 → **逐项报价抽屉** |
| 直选 | **报价选品卡** |

**一句话报价**：可跳过本卡（见文档 03 第四节）。

### 3.2 报价选品卡

- 推荐/筛选/规格规则同方案选品。  
- 「下一步：逐项报价」→ 生成待确认报价行（默认最新售价）→ **报价选品确认卡** 或进入逐项抽屉（直选可走确认卡再模板）。

### 3.3 报价选品确认卡

#### 内容

- 每行：规格、数量、最新/最低售价、**本单报价**输入。  
- 报价合计；直选可勾选「保存为方案」。

#### 交互

- 填价 → 「下一步：选择模板」→ **报价单模板抽屉**。  
- 行内改价/数量同步待确认报价明细。

### 3.4 逐项报价抽屉与语音条

#### 内容

- 按方案/直选/下单直选载入的行：规格、数量、最新/最低售价、本单报价。  
- 低于最低价行高亮警告。  
- 下单模式标题「逐项报价（下单）」；按钮「生成订单」。  
- **底部语音条**：按住说话 / 键盘；识别**不进对话区**。

#### 查询

- 待确认报价：行明细、是否下单填价模式、是否保存为方案、来源类型、关联方案编号

#### 交互（语音）

- 填价：「第一项报价 3680」「伺服电机 报价 4200」。  
- 改数量/规格。  
- 报价模式：「选择模板」→ 模板抽屉。  
- 下单填价模式：「生成订单」→ **下单确认抽屉**。

### 3.5 报价单模板抽屉

- 单选一种报价单版式 → 确认 → **报价单卡**。

### 3.6 报价单卡

#### 内容

- 报价号、客户、模板名、**报价金额**。  
- 「看文件」「生成订单」。

#### 查询

- 当前客户报价单

#### 交互

- **生成订单** → **下单确认抽屉**（沿用报价行与金额）。  
- **不**自动创建订单记录直至下单抽屉确认。

## 四、确认下单

### 4.1 下单来源卡

| 来源 | 条件 | 下一步 |
|------|------|--------|
| 按报价单 | 本客户已有报价单 | 直接 **下单确认抽屉** |
| 直接选品 | 已选客户 | **订单选品卡** |

### 4.2 订单选品卡

- 推荐/筛选/规格/数量同报价直选。  
- 「下一步：逐项报价」→ 下单填价模式 → **逐项报价抽屉**。  
- 可选「保存为方案」勾选。  
- 直选下单**必须**先逐项填价（无「按报价单跳过」除非已有报价单）。

### 4.3 下单确认抽屉

#### 内容

- 客户、来源摘要（报价单号+金额 **或** 直选行数+合计）。  
- 可选展示交期说明（本版主链路不强制交期）。

#### 交互

- **确认下单** → 写入演示订单库（状态**待排产**，含行明细、业务员）→ 关闭抽屉 → **订单成功卡**。

### 4.4 订单成功卡

- 订单号、客户、金额；只读。
