# 标注文档 02：方案速配 · 产品报价 · 确认下单

> **范围**：本版主链路三能力 + 共用选客户抽屉。  
> **原型**：演示数据 + 应用脚本；正式环境以各模块「查询逻辑」为准。  
> **前置**：方案/报价/下单多数步骤须已选**当前客户**（顶栏）；缺客户时出选客户引导卡。  
> **落页**：确认后按第八节映射表写入 `annotation-spec-data.js`。  
> **面板三段**（与 `test.md`、侧栏一致）：**① 内容 → ② 查询逻辑 → ③ 交互逻辑**。落页写入 `annotation-spec-data.js` 的 `content` / `query` / `interaction`。

---

## 一、整体结构

### 1.1 主链路（自上而下）

```
选定当前客户
  → 方案速配：选品改规格 → 方案预览改数量 → 方案模板 → 方案卡（无单价）
  → 产品报价：按方案 / 直选品 → 逐项报价 → 报价单模板 → 报价单卡
  → 确认下单：按报价单 / 直选品（须逐项填价）→ 下单确认抽屉 → 订单成功卡
```

### 1.2 原则

| 原则 | 说明 |
|------|------|
| 方案无价格 | 方案卡、方案选品**不展示单价**；金额仅在报价及之后 |
| 报价≠下单 | 生成报价单**不**自动建单；须报价单卡「生成订单」或技能「确认下单」 |
| 话术唯一入口 | 各模板/逐项抽屉**无**底部输入条；填价、选模板等在**输入区**发送（见文档 05） |
| 跨功能 | 切换主流程时询问是否沿用数据；不沿用则重选客户（文档 05 §2.3.2） |
| 一句话捷径 | 可跳过中间卡片（文档 03 第三、四节） |
| 历史卡片失效 | 弹出新助手业务卡后，上一条卡内按钮失效（文档 03 第二节） |

### 1.3 触发后出现

选客户引导卡、各入口/来源/选品/预览/选择卡、方案卡、报价单卡、逐项报价抽屉、报价单模板抽屉、方案模板抽屉、下单确认抽屉、订单成功卡、版式预览全屏层。

## 二、共用 · 选客户

### 模块 2.1：选客户抽屉

1. 底部抽屉；入口：顶栏切换客户、选客户引导卡「选择客户」
2. 标签页：客户 | 供应商 | 供应商/客户（默认「供应商/客户」排第一）
3. 搜索：名称、编码、结算客户（可不连续关键字）
4. 列表行：名称、编码、分类、新/老客户徽章
   - **查询逻辑**  
     - 老客户入列：本企业有订单且责任人 = 当前业务员（演示为「王业务」）。  
     - 公海新客户入列：无责任人且跟进状态 = 待跟进。  
     - 他人分管、非待跟进公海等不入列。  
   - **交互逻辑**  
     - 点行 → 写入当前客户并关闭抽屉。  
     - × 或点遮罩 → 不切换客户。  
     - 话术中带客户名：唯一命中直切；多家命中开抽屉并预填搜索；未命中提示核对名称（文档 05 §2.3.3）。

## 三、方案速配

### 模块 3.1：方案入口选择卡

1. 展示当前客户名；两条全宽操作：**查看历史数据** | **创建新方案**
   - **查询逻辑**：本客户已保存方案列表（查看历史用）。
   - **交互逻辑**  
     - 欢迎区/技能条「方案速配」→ 输入区发送入口口令「配个方案」→ 本卡。  
     - **查看历史** → 历史方案列表 → 点行出方案卡（版式预览须点卡内「预览 PDF」）。  
     - **创建新方案** → 清空方案流程草稿；新客户先**待补充需求提示卡**，老客户直接**方案选品卡**。  
     - 待跟进「做方案」→ 已绑定客户，**跳过本卡**直达创建路径。  
     - 一句话需求 → 直接方案卡（文档 03 §三）。

### 模块 3.2：待补充需求提示卡

1. 仅**新客户**创建方案路径；**无卡片内输入框**
2. 引导用户在对话区发送一条消息作为采购需求
   - **查询逻辑**：需求正文写入方案流程草稿；选品推荐区用**本次需求**匹配（不单用档案需求提示）。
   - **交互逻辑**  
     - 用户发送需求句 → 一条用户气泡 → 出方案选品卡。  
     - 选品卡上「修改需求」→ 重新出本提示卡，再发一条气泡覆盖需求。

### 模块 3.3：方案选品卡

写法与 `test.md` §2.2 一致，落页见 `card-plan-pick`。

### 模块 3.4：方案预览卡

1. 标题「方案预览 · 改规格」；展示已勾选行：品名、规格下拉、**购买数量**
2. 按钮：「返回选品」「生成方案」
   - **交互逻辑**  
     - 购买数量与规格在本卡确认（选品卡不做最终数量确认）。  
     - **返回选品** → 方案选品卡。  
     - **生成方案** → 打开**方案模板抽屉**（非直接出方案卡）。

### 模块 3.5：方案模板抽屉

1. 单选一种方案模板（序号 + 名称 + 描述）
2. 底部：「保存方案」
   - **交互逻辑**  
     - 点列表行选中模板。  
     - 点「保存方案」→ 对话区确认句「保存方案」→ 写入当前客户方案 → 关闭抽屉 → 插入**方案卡**。  
     - 输入区可说「第 1 个」「模板名称」「保存方案」（抽屉无底部输入条）。  
     - 未选模板时轻提示。  
     - 方案行：品名·规格·数量；**无价格**。

### 模块 3.6：方案卡

1. 方案编号、客户、模板名、行明细（品名 · 规格 · 数量）
2. 按钮：「预览 PDF」「去报价」
   - **查询逻辑**：当前客户已保存方案。
   - **交互逻辑**  
     - **预览 PDF** → 全屏技术方案版式预览（网页排版模拟）；表头序/品名/规格/数量/单位；页脚合计数量，**无单价与金额**。  
     - **去报价** → 无方案时提示先完成方案速配；有方案 → 载入方案行 → **逐项报价抽屉**（多方案时方案卡带编号可直进，不再出选择方案卡）。  
     - **无**「生成订单」按钮。

### 模块 3.7：方案版式预览层

1. 全屏层；顶栏「方案预览」+ 下载 + 转发 + 返回
   - **查询逻辑**：与当前会话方案对象一致。
   - **交互逻辑**：下载优先生成文件，离线时降级网页；转发为系统分享或复制摘要。

## 四、产品报价

### 模块 4.1：报价入口选择卡

1. **查看历史报价单** | **新建报价**
   - **交互逻辑**  
     - 技能条/欢迎区/说「报价」→ 本卡。  
     - 查看历史 → 历史报价单列表 → 点行出报价单卡（「看 PDF」在卡内）。  
     - 新建 → 无已保存方案则直进**选品报价卡**；有方案则**报价来源卡**。  
     - 说「按方案报价」→ **不经过本卡**，直达按方案流程（文档 05）。

### 模块 4.2：报价来源卡

1. 「按方案报价」「直接选品报价」；有方案时提示方案数量
   - **查询逻辑**：本客户已保存方案列表。
   - **交互逻辑**  
     - **无已保存方案**：不展示本卡，直进选品报价卡。  
     - **按方案报价**：仅 1 个方案 → 逐项报价抽屉；多个 → **选择方案卡**（主标题为方案名称）→ 再逐项或一句话出报价单卡。  
     - **直接选品报价** → 选品报价卡。

### 模块 4.3：选择方案卡（办理）

1. 列表展示本客户方案（方案名称为主）
   - **交互逻辑**：点行 → 对话区确认选方案 → 逐项报价抽屉。查看模式只出方案卡，不进入本办理流程（与历史查看区分见文档 03 / 填槽需求 §6.2）。

### 模块 4.4：选品报价卡

1. 推荐/筛选/规格规则同方案选品卡
2. 「下一步：逐项报价」
   - **交互逻辑** → 生成待确认报价行（默认最新售价）→ 逐项报价抽屉或报价选品确认卡（直选路径）。

### 模块 4.5：报价选品确认卡（直选路径可选）

1. 每行：规格、数量、最新/最低售价、**本单报价**；合计；可勾选「保存为方案」
   - **交互逻辑**  
     - 行内改价/数量同步待确认明细。  
     - 「下一步：选择模板」→ **报价单模板抽屉**。

### 模块 4.6：逐项报价抽屉

1. 载入方案行或直选行：规格、数量、最新/最低售价、本单报价；低于最低价行高亮
2. 报价模式按钮：「下一步：选择模板」；下单模式按钮：「确认下单」（见第五节）
3. **无抽屉底部输入条**
   - **查询逻辑**：待确认报价明细、是否下单填价模式、是否保存为方案、来源类型、关联方案编号。
   - **交互逻辑**  
     - 表内直接改数字 → 无用户气泡。  
     - 输入区填价：「第一项报价 4200」等 → 一条用户气泡后写入对应行。  
     - 点「下一步：选择模板」→ 对话区确认句 → 报价单模板抽屉。  
     - 下单模式点「确认下单」→ 下单确认抽屉。

### 模块 4.7：报价单模板抽屉

1. 单选报价单版式；确认生成
   - **交互逻辑** → 对话区「生成报价单」→ 关闭抽屉 → **报价单卡**。

### 模块 4.8：报价单卡

1. 报价单号、客户、模板名、**报价金额**；「看 PDF」「生成订单」
   - **查询逻辑**：当前客户报价单。
   - **交互逻辑**  
     - **看 PDF** → 全屏销售报价单预览；含单价/金额；页脚含税合计。  
     - **生成订单** → **下单确认抽屉**（沿用报价行与金额）；**不**自动建单直至下单确认提交。  
     - 多报价单时卡带编号，下单不再弹出选择报价单卡。

### 模块 4.9：报价单版式预览层

同模块 3.7；顶栏为「报价单预览」；数据与当前报价单一致。

## 五、确认下单

### 模块 5.1：下单来源卡

1. 「按报价单」「直接选品」
   - **查询逻辑**：本客户已保存报价单列表。
   - **交互逻辑**  
     - **无报价单**：不展示本卡，直进选品报价卡（下单模式）。  
     - **按报价单**：仅 1 张 → 下单确认抽屉；多张 → **选择报价单卡**。  
     - **直接选品** → 选品报价卡（下单模式）。  
     - **不支持**按方案直接下单。

### 模块 5.2：选择报价单卡（办理）

1. 列表展示本客户报价单
   - **交互逻辑**：点行 → 对话区确认按该报价单下单 → 下单确认抽屉。

### 模块 5.3：选品报价卡（下单模式）

1. 规则同模块 4.4；「下一步：逐项报价」→ 逐项报价抽屉（下单填价模式）；可勾选「保存为方案」
   - **交互逻辑**：直选下单**必须**先逐项填价（无报价单时不能跳过）。

### 模块 5.4：下单确认抽屉

1. 客户、来源摘要（报价单号+金额 **或** 直选行数+合计）；可选交期说明（本版主链路不强制）
2. 「确认下单」
   - **交互逻辑**  
     - 确认 → 写入演示订单（状态待排产，含行明细、业务员）→ 关闭抽屉 → **订单成功卡**。  
     - 提交前必填校验见文档 04。

### 模块 5.5：订单成功卡

1. 订单号、客户、状态、订单明细（品名、规格、数量、单位）、合计金额
   - **交互逻辑**：只读展示。
