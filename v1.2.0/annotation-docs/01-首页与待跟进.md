# 标注文档 01：对话首页与待跟进

> **范围**：登录后「对话页」首屏 + 待跟进全链路（含写跟进子步）。  
> **原型**：演示数据 + 应用脚本；正式环境以各模块「查询逻辑」为准。  
> **落页**：确认后按第七节映射表写入网页标注数据。

---

## 一、整体结构

### 1.1 四大类

| 大类 | 包含模块 | 常驻 |
|------|----------|------|
| 一、对话壳 | 顶栏、当前客户条、消息区、技能条、输入区 | 是 |
| 二、首屏默认区 | 欢迎区、模板消息卡、待跟进摘要卡、最近访问 | 首屏写入消息流 |
| 三、待跟进业务流 | 列表卡、企业详情卡、下一步引导卡、写跟进抽屉 | 按操作出现 |
| 四、大模型能力 | 意图分配、场景回复 | 贯穿输入区与技能条 |

### 1.2 首屏默认可见（自上而下）

顶栏 → 当前客户条 → 欢迎区 →（服务号模板入口时）模板消息卡 → 待跟进摘要卡 → 最近访问 → 技能条 → 输入区。

### 1.3 待跟进子模块

待跟进摘要卡、待跟进列表卡、企业详情卡、下一步引导卡、写跟进抽屉。  
列表为**今日待跟进企业列表**（新客户与老客户合并，无分标签页筛选），行内徽标区分归属；**待跟进企业数量** = 列表行数。

### 1.4 触发后出现

待跟进列表卡、企业详情卡、下一步引导卡、写跟进抽屉、选客户抽屉、版式预览层（方案/报价单，见文档 02）。

## 二、对话壳

### 模块 2.1：顶栏

1. 标题：「销售助手 · Pro」（仅智能体名，无企业切换入口）

### 模块 2.2：当前客户条

1. 未选显示「请选择客户」；已选显示客户全称（可截断）
2. 已选时展示：客户性质 + 新/老客户徽标（判定见第六节）
   - **查询逻辑**：来往单位主数据；选客户抽屉标签页为**客户 / 供应商 / 供应商/客户**，支持名称、编码、结算客户搜索（可不连续关键字）。
   - **交互逻辑**：点击 → 打开选客户抽屉 → 选中 → 轻提示「已切换至某某」→ 刷新顶栏；**不**自动打开企业详情卡。切换客户后清空方案/报价/下单流程草稿（跨功能选择「沿用数据」时除外）。
3. **清空记录**（标注钉：清空记录按钮）：二次确认后删除消息区**全部**记录（含欢迎区、待跟进摘要、最近访问及后续对话），并清空本地会话缓存；同日不自动重发每日首屏，次日首次进入再发。

### 模块 2.3：消息区

1. 可滚动对话区：用户气泡、智能助手气泡、业务卡片、系统提示
   - **查询逻辑**：按当前企业保存会话；单企业最多保留 200 条（演示）。
   - **交互逻辑**：新消息滚到底部；**每弹出一张新的助手业务卡片**，上一条消息卡片序号内全部按钮失效（细则见文档 03 §二）。用户仅发送文字、尚未弹出新卡片时，当前卡片按钮仍可用。

### 模块 2.4：技能条

1. 输入区上方横滑条：待跟进、方案速配、产品报价、确认下单、交期、复制订单、变更、查进度、客服工单等（演示均可点）
   - **查询逻辑**：当前激活能力名称与底栏高亮同步。
   - **交互逻辑**：点击某项 → 高亮该项 → 按该能力默认口令进入对应流程（待跟进 → 今日待跟进列表；方案/报价/下单见文档 02）。切换至另一主流程且已有可沿用数据时，弹出跨功能确认（文档 02 / 文档 05）。

### 模块 2.5：输入区

1. 文字输入 + 发送；按住说话松手转文字后发送（演示）
   - **交互逻辑**：**唯一话术入口**；方案/报价类抽屉**无**底部输入条，待填话术均在本区发送（意图与填槽见文档 05；通道细则见《用户输入与意图填槽需求》）。

## 三、首屏默认区

> 首屏由「每日首次进入」写入消息区（一条助手气泡内可含多块）。

**消息区 · 会话与刷新规则**

| 场景 | 规则 |
|------|------|
| 每日首次进入 | 每自然日、每企业第一次打开：欢迎区 + 待跟进摘要 + 最近访问 |
| 刷新页面 | 同日再次进入：从本地恢复完整消息流 |
| 重新登录 | 清空会话与每日标记，重新首屏 |
| 清空记录 | 删除全部消息与缓存；不自动重发首屏 |
| 服务号模板入口 | 左侧微信区展示模板消息卡；点按钮进入演示页并拉待跟进列表 |

### 模块 3.1：欢迎区

1. 主文案：「你好，我是销售助手。可点下方待跟进查看企业，或使用底部技能完成方案、报价、交期、下单等操作。」
2. 副文案：列出支持能力名称
   - **查询逻辑**：静态文案。
   - **交互逻辑**：首屏第一条智能助手内容；无点击。

### 模块 3.2：待跟进摘要卡

1. 标题「今日待跟进」；展示**待跟进企业数量**（如「3 家企业」）；引导「点击查看列表」
   - **查询逻辑**：调用今日待跟进查询，得到列表与数量；与首屏初始化同时拉数（规则见第六节）。
   - **交互逻辑**：点击 → 对话区出现用户确认句「今日待跟进」→ 约 300 毫秒后回复待跟进列表卡。数量为 0 时显示「0 家企业」，可点击，列表为空。

### 模块 3.3：最近访问

1. 标题「最近访问」；项为「客户名 · 时间标签」，当前企业下最多 3 条（演示）
   - **查询逻辑**：正式环境为最近访问记录，按时间倒序取前三。
   - **交互逻辑**：紧接摘要卡后插入。点击某项 → 切换当前客户 + 技能条对应能力高亮 → 进入该能力（待跟进 → 列表卡；方案/报价等见文档 02）。

### 模块 3.4：模板消息卡（服务号 · 待跟进）

1. 渠道标签「服务号消息」；标题「今日待跟进提醒」；正文含待跟进企业数量 N；按钮「查看待跟进列表」
   - **查询逻辑**：模板文案由服务号配置；N 与今日待跟进查询一致。
   - **交互逻辑**：用户在服务号点击 → 进入演示页 → 点按钮 → 对话区用户句「今日待跟进」→ 回复待跟进列表卡（与摘要卡、技能条「待跟进」一致）。

## 四、待跟进业务流

### 模块 4.1：待跟进列表卡

1. 标题「今日待跟进 · X 家」；行：企业名称 + 新/老客户徽标
   - **查询逻辑**：与摘要卡共用同一份今日待跟进企业列表与数量；排序见 6.4。
   - **交互逻辑**：点击摘要、话术/技能「待跟进」、意图识别为今日待跟进时，在消息流**追加**列表卡（可多次）。点击行 → 设当前客户（静默）→ 追加企业详情卡 + 下一步引导卡。

### 模块 4.2：企业详情卡

1. 标题「企业详情」+ 新/老客户徽标
2. 字段：编码、名称、性质、客户性质、结算客户、客户等级、联系人地址（姓名·电话·地址）
   - **查询逻辑**：按客户标识取主数据/详情接口。
   - **交互逻辑**：从列表或最近访问选企业后追加；只读。

### 模块 4.3：下一步引导卡

1. 标题「接下来您想做什么？」；按钮：写跟进 | 做方案 | 稍后再说
   - **交互逻辑**：紧接详情卡追加。**写跟进** → 模块 4.4；**做方案** → 切换客户后进入方案速配创建路径（文档 02，不经方案入口「查看历史」分支）；**稍后再说** → 智能助手固定短句，不打开抽屉。

### 模块 4.4：写跟进抽屉

1. 字段：联系人*、联系方式*、发货地址、跟进信息*、跟进时间*、跟进状态（跟进中 / 跟进完成）
2. 表单「跟进状态」用于本次提交；列表「新客户」判定中的「待跟进」指**历史最新一条跟进记录**状态，二者不同。
   - **查询逻辑**：正式环境提交跟进接口，关联客户、企业、跟进人；演示为前端校验。
   - **交互逻辑**：有当前客户时打开，标题「写跟进 — {企业名}」，默认带入联系人信息。无当前客户时意图为写跟进 → 先出选客户引导卡。抽屉**无**底部话术条；补充说明在输入区发送。提交成功 → 关闭 → 用户消息「已提交跟进记录」→ 智能助手确认。× 或遮罩关闭。

## 六、新客户 / 老客户待跟进查询

> 用于待跟进摘要数量、列表行、新/老客户徽标。输出**今日待跟进企业列表**（合并，无分 tab）与**待跟进企业数量**。

### 6.1 总体

- **范围**：当前登录用户、当前企业下往来单位。
- **合并**：老客户待跟进 ∪ 新客户待跟进；每家单位仅属一类。
- **判定顺序**：先判老客户；否则判新客户。均不满足则不进入列表。

### 6.2 老客户（须同时满足）

1. **分管关系**：往来单位分管人员 = 当前登录用户。
2. **超时未下单**：当前日期 − 最近有效订单日期 > 主数据「销售提醒天数」（无订单日期口径以企业主数据配置为准）。

归入列表，徽标「老客户」。

### 6.3 新客户（须同时满足）

1. **无分管**：分管人员为空或公海。
2. **跟进状态**：最新一条跟进记录状态 =「待跟进」（与写跟进表单「跟进中/完成」不同）。无跟进记录则不归入新客户。

归入列表，徽标「新客户」。

### 6.4 合并、排序、条数与刷新

1. 查老客户 → 查新客户 → 合并 → 计数。
2. **排序**：最近更新时间倒序；同时间可按名称升序。
3. **条数**：一次返回全量；摘要与列表数量一致。
4. **拉数时机**：首屏、展开列表、切换企业、写跟进提交后（是否刷新以接口为准）。
5. **接口失败**：轻提示；数量可显示上次结果或「—」。

### 6.5 原型说明

演示数据按企业取前若干条；徽标用演示「客户类型」字段，不用于正式验收口径。
