## 二、对话流 · 历史页面失效

### 2.1 规则说明

1. **触发条件**：对话区**每弹出一个新的助手页面**（插入一条新的助手气泡，且为一张业务卡或等价整页内容；`pushAiHtml` 默认触发）。
2. **失效范围**：失效**上一页面序号**内所有助手气泡上的操作按钮（同序号多条气泡一并失效）。
3. **不触发**：用户**仅发送**文字、语音气泡，且**尚未**弹出下一张助手页时，当前页按钮仍可用。
4. **同页补充**：筛选提示、已选品说明等 `samePage` 补充气泡与当前页共用序号，**不**使当前页按钮失效。
5. **仍可点击**：**最新页面序号**内的按钮；用户气泡、系统提示、抽屉/弹窗内操作不受影响。
6. **点击已失效按钮**：轻提示「该页面已过期，请使用最新页面中的操作」；按钮禁用并呈半透明样式。

### 2.2 与消息区关系

- 首屏欢迎区、待跟进摘要、最近访问**合并为一条助手消息**（一个气泡内三块），共用页面序号 1；三块之间**互不失效**。
- **弹出下一张助手页**（如待跟进列表、选品卡）后，整条首屏（含功能格、待跟进、最近访问）一并失效。

### 2.3 原型实现要点

- `pushAiHtml(html)` 弹出前调用 `advanceFlowPage()`；`pushAiHtml(html, { samePage: true })` 不递增。
- `pushUserMsg` 不触发失效。
- 标注面板不展示样式计算值；验收只看行为与面板文案。

## 三、方案速配 · 一句话生成方案

**话术格式**：可一句话生成方案（话术含选品、规格与数量）。示例：给深圳创源配伺服电机和传动齿轮箱各2台

### 3.1 适用条件

| 项 | 规则 |
|----|------|
| 前置 | 已选当前客户 |
| 话术 | **描述性需求句**，非纯口令（如非单独「配个方案」「选品 xxx」「加购」） |
| 识别 | 含「需要/想要/各N台/电机/轴承…」等 **或** 长度≥12 且非「配个/报价」开头；**且** 不含报价类关键词（见第四节） |
| 排除 | 「报价/报个价/折扣…」类句子走报价一句话 |

### 3.2 处理逻辑（查询）

1. 按客户与话术匹配产品：新客户先过客户性质×产品类型候选池，再用需求描述关键词与存货描述算相似度；话术里提到的品名也会参与；若没有明显命中，则用推荐区前 4 条顶替。
2. 解析数量（「各 2 台」「品名…10台」）。
3. 写入方案流程上下文（已选品、规格、数量）；默认使用第一种标准技术方案模板。
4. 直接生成**方案卡**，**跳过**选品卡、购物车、模板抽屉。

### 3.3 交互

1. 用户发送需求句 → 用户气泡 → 引导句「已根据描述自动匹配 N 项…」→ **方案卡**（含文件预览、去报价）。
2. 匹配失败：轻提示 + 仍进入**分步选品**，筛选框预填话术片段。
3. 与分步流程并存：说「配个方案」仍走选品 → 购物车 → 模板。

## 四、产品报价 · 一句话生成报价单

**话术格式**：可一句话生成报价单（话术含按方案或选品、规格、数量、单价或折扣）。示例：给深圳创源按方案报价打九折

### 4.1 适用条件

| 项 | 规则 |
|----|------|
| 前置 | 已选客户 |
| 话术 | 含 **单价/折扣/打N折** 或较长「按方案报价…」描述；**不含**单独口令「去报价」「报价」「按方案报价」 |
| 入口口令 | 「去报价」「报价」「按方案报价」等短句 → 与方案卡按钮一致，打开**逐项报价**（有方案则载入方案行），**不**直出报价单卡 |
| 分支甲 | 已有方案 + 一句话含折扣/单价等 → 按方案行直出报价单卡 |
| 分支乙 | 话术含报价意图 + 产品描述 → 按描述匹配产品行，填价后出卡 |

### 4.2 处理逻辑（查询）

1. **按方案**：载入方案行；解析折扣（九折、九五折、折扣 15 等）作用于行单价。
2. **按描述**：匹配产品；单价优先从话术解析，否则用最新售价参考；再应用折扣。
3. 使用默认报价单版式；写入当前客户报价单；**不打开**逐项报价抽屉与模板抽屉。

### 4.3 交互

1. 引导句说明已匹配品名 / 是否按方案 / 是否打折 → **报价单卡**（看文件、生成订单）。
2. 说单独「报价」且无法匹配产品时：轻提示 + 出 **报价来源卡**。
3. 与分步报价并存：来源卡 → 选品 → 逐项报价 → 模板 → 报价单卡。

## 四之二、确认下单 · 一句话生成订单

**话术格式**：可一句话生成订单（话术含按报价单或直选品名、数量与单价）。示例：给华东精密下单，配伺服电机10台单价2100

- **按报价单**：话术含「按报价单」等，且本客户已有报价单 → 打开下单确认。
- **直选**：话术含品名、数量、单价 → 匹配后打开下单确认。

## 五、方案速配 · 选品「更多产品」懒加载

### 5.1 内容

1. **推荐区**（上）：老客户历史订单前 10 条 / 新客户需求匹配前 10 条（规则见方案选品卡查询）。
2. **更多产品**（下）：演示产品库中**不在推荐区**的品项；支持筛选框过滤品名/规格/存货描述。
3. 首屏「更多」仅渲染 **5 条**；底部文案「已显示 x / 总数，继续下滑加载（每页 5 条）」。
4. 在选品列表滚动容器内下滑，触底再加载 **5 条**，直至「已加载全部 N 条」。

### 5.2 查询逻辑

**更多产品候选（`planMoreProducts`）**

| 客户类型 | 排除集 | 纳入条件 | 排序 |
|----------|--------|----------|------|
| 老客户 | 推荐区已展示的产品 | 搜索框有字：品名/规格/存货描述包含该字；无字：余量全展示 | 品名升序 |
| 新客户 | 同上（须在类型候选池内） | 需求描述关键词与「存货描述」相似度 20%～80%；或搜索框有字且品名/规格/描述包含 | 相似度从高到低 |

- 推荐区新客户：先客户性质×产品类型，再需求描述关键词与「存货描述」算相似度，须 **超过 80%**。  
- 分页：`moreVisible` 默认 5；筛选或语音「筛选」后重置为 5；筛选同时作用于推荐区与更多区。  
- 触底哨兵进入视口再加载 5 条，直至全部展示。

### 5.3 交互逻辑

- 懒加载**不改变**已选规格/勾选状态（刷新整张选品卡时从方案流程上下文恢复）。
- 加购、生成方案、语音选品与懒加载无冲突。

## 六、演示主数据说明（方案选品用）

方案选品推荐区：老客户走历史订单；新客户先客户性质×产品类型，再需求描述关键词×存货描述。**待跟进列表/写跟进规则属上一版**，不在本版标注范围。

| 字段含义 | 方案速配用途 |
|----------|----------------|
| 老客户 | 有过订单且责任人=当前用户；推荐区=历史订单产品；更多区=全库余量+搜索框过滤 |
| 新客户 | 待跟进且无责任人；推荐区先类型匹配再描述匹配 |
| 客户性质 | 客户档案 `category`（终端客户/经销商/战略客户），决定可推荐的产品类型范围 |
| 产品类型 | 产品主数据 `productType`（如机械传动、自动化），须落在客户性质对应范围内 |
| 需求描述关键词 | 客户档案 `demandHint`，第二步与存货描述算相似度 |
| 存货描述 | 产品主数据 `inventoryDesc`，第二步匹配字段 |
| 责任人 | 老客户须等于当前业务员；新客户须为空 |

## 八、验收抽验建议

1. 选客户 →「配个方案」弹出选品卡 → 再弹出下一张助手页（如购物车）→ 选品卡按钮变灰；仅发文字且未弹新页时选品卡仍可用。  
2. 「客户需要伺服电机和传动齿轮箱各2台」→ 直接出方案卡。  
3. 「给深圳创源报伺服电机10台打九折」→ 直接出报价单卡。  
4. 选品卡「更多产品」下滑加载，每批 5 条。
