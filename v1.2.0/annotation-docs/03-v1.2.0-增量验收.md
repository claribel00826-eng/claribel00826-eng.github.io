## 二、对话流 · 历史操作按钮失效

### 2.1 规则说明

1. **触发条件**：用户在对话区**每发送一条有效新消息**（含键盘发送、语音转写后路由、技能条/欢迎区触发的用户气泡、待跟进「今日待跟进」等写入用户消息的路径）。
2. **失效范围**：失效**上一步**（流程步序号当前值）对应的所有助手气泡内、带操作标识的**按钮**与选品行按钮。
3. **与功能无关**：不要求新消息切换功能；同一功能连发两条（如先「配个方案」再「加购」）也会使上一条助手卡内按钮不可点。
4. **仍可点击**：**最新一步**助手回复中的按钮；用户气泡、系统提示无按钮不受影响。
5. **恢复历史会话**：从服务端数据库恢复的助手卡，其按钮**一律失效**；新会话从第 1 步开始。
6. **点击已失效按钮**：轻提示「已有新消息，请使用最新对话中的操作」；按钮禁用并呈半透明样式。

### 2.2 与消息区关系

- 首屏欢迎区、待跟进摘要、最近访问**合并为一条助手消息**（一个气泡内三块内容），共用流程步第 1 步；用户**首次**发消息后该条内全部按钮一并失效（含功能格、待跟进展开、最近访问）。
- 此后每条新助手回复单独占一步；用户下一条消息前该步为「当前可操作步」。

### 2.3 原型实现要点

- 流程步序号仅保存在当前会话，不持久化；发送用户消息前递增步序号。
- 标注面板不展示样式计算值；验收只看行为与面板文案。

## 三、方案速配 · 一句话生成方案

**话术格式**：可一句话生成方案（话术含选品、规格与数量）。示例：给深圳创源配伺服电机和传动齿轮箱各2台

### 3.1 适用条件

| 项 | 规则 |
|----|------|
| 前置 | 已选当前客户 |
| 话术 | **描述性需求句**，非纯口令（如非单独「配个方案」「选品 xxx」「加购」） |
| 识别 | 含「需要/想要/各N台/电机/轴承…」等 **或** 长度≥12 且非「配个/报价」开头；**且** 不含报价类关键词（见第四节） |
| 排除 | 「报价/报个价/折扣…」类句子走报价一句话 |

### 3.2 处理逻辑（查询）

1. 按客户与话术匹配产品：品名、存货描述、新客户需求关键词，得分达阈值；无强匹配时回退推荐列表前 4 条。
2. 解析数量（「各 2 台」「品名…10台」）。
3. 写入方案流程上下文（已选品、规格、数量）；默认使用第一种标准技术方案模板。
4. 直接生成**方案卡**，**跳过**选品卡、购物车、模板抽屉。

### 3.3 交互

1. 用户发送需求句 → 用户气泡 → 引导句「已根据描述自动匹配 N 项…」→ **方案卡**（含文件预览、去报价）。
2. 匹配失败：轻提示 + 仍进入**分步选品**，筛选框预填话术片段。
3. 与分步流程并存：说「配个方案」仍走选品 → 购物车 → 模板。

## 四、产品报价 · 一句话生成报价单

**话术格式**：可一句话生成报价单（话术含按方案或选品、规格、数量、单价或折扣）。示例：给深圳创源按方案报价打九折

### 4.1 适用条件

| 项 | 规则 |
|----|------|
| 前置 | 已选客户 |
| 话术 | 含 **报价/报个价/单价/折扣/打N折/按方案** 等 |
| 分支甲 | 已有本客户方案，且话术为「按方案/方案报价」或报价句中**无强产品匹配** → 按方案行生成 |
| 分支乙 | 话术含报价意图 + 产品描述 → 按描述匹配产品行，填价后出卡 |

### 4.2 处理逻辑（查询）

1. **按方案**：载入方案行；解析折扣（九折、九五折、折扣 15 等）作用于行单价。
2. **按描述**：匹配产品；单价优先从话术解析，否则用最新售价参考；再应用折扣。
3. 使用默认报价单版式；写入当前客户报价单；**不打开**逐项报价抽屉与模板抽屉。

### 4.3 交互

1. 引导句说明已匹配品名 / 是否按方案 / 是否打折 → **报价单卡**（看文件、生成订单）。
2. 说单独「报价」且无法匹配产品时：轻提示 + 出 **报价来源卡**。
3. 与分步报价并存：来源卡 → 选品 → 逐项报价 → 模板 → 报价单卡。

## 四之二、确认下单 · 一句话生成订单

**话术格式**：可一句话生成订单（话术含按报价单或直选品名、数量与单价）。示例：给华东精密下伺服电机10台单价2100

- **按报价单**：话术含「按报价单」等，且本客户已有报价单 → 打开下单确认。
- **直选**：话术含品名、数量、单价 → 匹配后打开下单确认。

## 五、方案速配 · 选品「更多产品」懒加载

### 5.1 内容

1. **推荐区**（上）：老客户历史订单前 10 条 / 新客户需求匹配前 10 条（规则见方案选品卡查询）。
2. **更多产品**（下）：演示产品库中**不在推荐区**的品项；支持筛选框过滤品名/规格/存货描述。
3. 首屏「更多」仅渲染 **5 条**；底部文案「已显示 x / 总数，继续下滑加载（每页 5 条）」。
4. 在选品列表滚动容器内下滑，触底再加载 **5 条**，直至「已加载全部 N 条」。

### 5.2 查询逻辑

**更多产品候选（`planMoreProducts`）**

| 客户类型 | 排除集 | 纳入条件 | 排序 |
|----------|--------|----------|------|
| 老客户 | 推荐区已展示的品项 ID | 筛选词命中品名/规格/存货描述；无筛选词则余量全展示 | 品名升序 |
| 新客户 | 同上 | `demandHint` 与 **存货描述字段** 匹配分 ∈ (0.2, 0.8]；或有筛选词且三字段命中 | 匹配分降序 |

- 推荐区新客户：同一 `demandHint` × `inventoryDesc`，分 **>0.8**（仅描述字段，品名不计分）。  
- 分页：`moreVisible` 默认 5；筛选或语音「筛选」后重置为 5；筛选同时作用于推荐区与更多区。  
- 触底哨兵进入视口再加载 5 条，直至全部展示。

### 5.3 交互逻辑

- 懒加载**不改变**已选规格/勾选状态（刷新整张选品卡时从方案流程上下文恢复）。
- 加购、生成方案、语音选品与懒加载无冲突。

## 六、演示主数据说明（方案选品用）

方案选品推荐区使用客户类型与需求关键词（新客户需求匹配）；**待跟进列表/写跟进规则属上一版**，不在本版标注范围。

| 字段含义 | 方案速配用途 |
|----------|----------------|
| 客户类型·老客户 | 推荐区：历史订单；更多区：全库余量 + 筛选 |
| 客户类型·新客户 | 推荐区与更多区均关联 demandHint × 存货描述 |
| 需求关键词（demandHint） | 新客户主数据中的需求词，与 **inventoryDesc** 计分 |
| 存货描述（inventoryDesc） | 新产品主数据字段，新客户匹配的唯一计分字段 |

## 八、验收抽验建议

1. 选客户 →「配个方案」→ 再发任意新消息 → 方案选品卡按钮变灰。  
2. 「客户需要伺服电机和传动齿轮箱各2台」→ 直接出方案卡。  
3. 「给深圳创源报伺服电机10台打九折」→ 直接出报价单卡。  
4. 选品卡「更多产品」下滑加载，每批 5 条。
