## 二、对话流 · 历史消息卡片失效

### 模块 2.1：规则说明

1. **触发**：对话区**每插入一条新的智能助手业务卡片**（一条助手气泡内含一张完整业务卡或等价整卡内容）。
2. **失效范围**：**上一条消息卡片序号**内，该序号下所有助手气泡上的操作按钮一并失效。
3. **不触发**：用户**仅发送**文字或语音气泡，且**尚未**弹出下一张业务卡片时，当前卡片按钮仍可用。
4. **同序号补充**：筛选提示、已选品说明等与当前卡同序号的补充气泡，**不**使当前卡按钮失效。
5. **仍可用**：**最新序号**内卡片按钮；用户气泡、系统提示、抽屉/弹窗内操作不受影响。
6. **点击已失效按钮**：轻提示「该消息卡片已过期，请使用最新消息卡片中的操作」；按钮置灰半透明。

### 模块 2.2：与首屏的关系

1. 欢迎区、待跟进摘要、最近访问**合并为一条助手消息**（一个气泡内多块），共用消息卡片序号 1；三块之间**互不失效**。
2. 弹出下一张业务卡（如待跟进列表、方案选品卡）后，整条首屏（含功能格、待跟进、最近访问）按钮一并失效。

### 模块 2.3：验收要点

- 验收只看行为与标注面板文案，不验收样式计算细节。  
- 标注钉「对话框 · 消息区」承载本节规则。

## 三、方案速配 · 一句话生成方案

**话术示例**：给深圳创源配伺服电机和传动齿轮箱各 2 台

### 模块 3.1：适用条件

1. **前置**：已选当前客户。
2. **话术**：描述性需求句（含需要/各 N 台/品名等），**非**单独口令「配个方案」「选品 xxx」。
3. **排除**：含报价、折扣类关键词 → 走报价一句话（第四节）。

### 模块 3.2：查询逻辑

1. 按客户与话术匹配产品：新客户先客户性质×产品类型，再需求关键词与存货描述相似度；话术品名参与；无明显命中可用推荐区前若干条顶替。
2. 解析数量（「各 2 台」等）。
3. 写入方案流程草稿（已选品、规格、数量）；默认第一种标准技术方案模板。
4. **跳过**选品卡、方案预览、模板抽屉，直接生成**方案卡**。

### 模块 3.3：交互逻辑

1. 用户发送需求句 → 用户气泡 → 引导句「已根据描述自动匹配 N 项…」→ **方案卡**（含版式预览、去报价）。
2. 匹配失败：轻提示 + 仍进入分步选品，筛选框预填话术片段。
3. 与分步并存：说「配个方案」→ 入口选择卡 →「创建新方案」→ 选品 → 预览 → 模板。

## 四、产品报价 · 一句话生成报价单

**话术示例**：给深圳创源按方案报价打九折

### 模块 4.1：适用条件

1. **前置**：已选客户。
2. **话术**：含单价/折扣/打 N 折，或较长「按方案报价…」描述；**不含**单独短口令「报价」「按方案报价」（短口令打开逐项报价，不直出报价单卡）。
3. **分支**：已有方案 + 含折扣/单价 → 按方案行直出报价单卡；含报价意图 + 产品描述 → 按描述匹配填价后出卡。

### 模块 4.2：查询逻辑

1. **按方案**：载入方案行；解析折扣作用于行单价。
2. **按描述**：匹配产品；单价优先话术解析，否则最新售价；再应用折扣。
3. 默认报价单版式；写入当前客户报价单；**不打开**逐项抽屉与模板抽屉。

### 模块 4.3：交互逻辑

1. 引导句说明匹配品名 / 是否按方案 / 是否打折 → **报价单卡**。
2. 单独「报价」且无法匹配产品：轻提示 + **报价来源卡**。
3. 与分步并存：来源卡 → 选品 → 逐项 → 模板 → 报价单卡。

## 五、确认下单 · 一句话生成订单（简述）

1. **按报价单**：话术含按报价单等，且本客户已有报价单 → 打开下单确认抽屉。  
2. **直选**：话术含品名、数量、单价 → 匹配后打开下单确认。  
3. 细则与缺槽见文档 04、文档 05。

## 六、方案速配 · 选品「更多产品」懒加载

### 模块 6.1：展示内容

1. **推荐区**（上）：规则见文档 02 模块 3.3。  
2. **更多产品**（下）：产品库中不在推荐区的品项；支持筛选框。  
3. 首屏「更多」仅 **5 条**；文案「已显示 x / 总数，继续下滑加载（每页 5 条）」。  
4. 列表容器内下滑触底，每次再加载 **5 条**，直至全部加载完毕。

### 模块 6.2：查询逻辑

| 客户类型 | 更多区候选 |
|----------|------------|
| 老客户 | 去掉推荐区已展示；有筛选词则品名/规格/描述包含该字；无词则余量全展示；品名升序 |
| 新客户 | 类型候选池内去掉推荐区；相似度 20%～80% 或筛选词命中；按相似度从高到低 |

- 推荐区新客户：类型匹配后，需求与存货描述相似度须 **>80%**。  
- 筛选或点「筛选」后，更多区分页重置为 5 条。

### 模块 6.3：交互逻辑

- 懒加载**不改变**已勾选与规格（刷新整张选品卡时从方案流程草稿恢复）。  
- 与加购、生成方案、选品勾选无冲突。

## 七、演示主数据（方案选品用）

客户类型（老客户/新客户）在**方案选品卡、待补充需求卡**验收，不在顶栏全局区重复挂规则。

| 字段含义 | 用途 |
|----------|------|
| 老客户 | 有订单且责任人=当前用户；推荐区=历史订单产品 |
| 新客户 | 待跟进且无责任人；推荐区先类型再描述匹配 |
| 客户性质 | 决定可推荐的产品类型范围 |
| 产品类型 | 须落在客户性质对应范围 |
| 需求描述关键词 | 与存货描述算相似度（新客户） |
| 存货描述 | 产品主数据描述字段 |
| 责任人 | 老客户=当前业务员；新客户=空 |

待跟进列表规则见**文档 01 第六节**，不在本版重复。

## 九、验收抽验建议

1. 选客户 →「配个方案」出选品卡 → 再出方案预览卡 → 选品卡按钮变灰；仅发文字未出新卡时选品卡仍可用。  
2. 「客户需要伺服电机和传动齿轮箱各 2 台」→ 直接出方案卡。  
3. 「给深圳创源报伺服电机 10 台打九折」→ 直接出报价单卡。  
4. 选品卡「更多产品」下滑，每批 5 条。
