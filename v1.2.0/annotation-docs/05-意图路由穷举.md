## 二、意图识别模型

### 2.1 识别结果（四项）

每句输入先解析四项（意图钉「业务说明」写逻辑，下表为源稿摘要）：

| 字段 | 逻辑（做什么） |
|------|----------------|
| **功能意图** | 识别用户要办的主能力；本句或来源卡若已点明走法，则记入该能力下的子路径（报价/下单的来源分支），否则只记主能力、分支待补 |
| **已读槽位** | 从本句抽出业务字段（客户、品名、方案名、报价单号等）并与会话合并；本句写明的覆盖旧值；定义见《数据填槽需求》 |
| **当前步骤** | 按当前技能与最新助手页，判断处于哪一步、哪张卡/抽屉 |
| **下一步** | 看本步必填是否已齐、以及按名称/编号查方案或报价单是「一条还是多条」，定系统动作类型（§2.3.1）；**不是**表单字段 |

**解析顺序**：功能意图 → 业务字段 → 对照当前步必填 → 下一步类型。  
**判定顺序**：先判跨功能（§2.2），否则在活跃卡上填本步字段。

### 2.2 判定优先级（当前界面 vs 跨功能）

**不冲突**：先判跨功能，未命中再判当前界面；同一句只走一条分支。

| 顺序 | 条件 | 处理 |
|------|------|------|
| ① | 本句指向**另一功能**，且上下文满足目标必填 | **切换功能卡**（询问是否带入） |
| ② | 否则，仍在当前功能内 | **当前界面**：以最新助手页填本步字段 |

例：在方案选品卡说「报价」→ 走 ①（若已有选品/方案）；说「筛选 伺服」→ 走 ②。

### 2.3 上下文共用与名称检索

- **上下文共用**：多轮合并业务字段（前句客户 + 后句品名等）；合并后齐备则 **推进**，仍缺则 **缺槽引导**。
- **名称检索**（方案名/报价单号等）：库里只命中一条 → **推进**；命中多条 → **消歧**（选择卡，可语音「第 N 条」）；对不上 → **缺槽引导**或出全量列表。
- **步骤跳转**：本步必填齐备可跳中间界面，归类为 **推进**。

### 2.3.1 下一步五类（逻辑）

| 类型 | 逻辑 |
|------|------|
| **推进** | 当前步该填的已齐，可进下一步；或方案/报价单已唯一确定；或在当前卡完成本步修改 |
| **消歧** | 方案/报价单命中多条，须先在列表选定（可语音「第 N 条」） |
| **缺槽引导** | 缺必填，或客户/品名/方案/报价单对不上记录 →【待填写】+ 打开对应界面 |
| **跨功能** | 改做另一能力且数据够目标起步 → 切换功能卡询问是否带入 |
| **能力说明** | 无法识别或点失效页 → 只说明能力或提示用最新页 |

### 2.3.2 跨功能切换（细则）

当输入语义属于**另一功能**（如在方案速配中说「报价」「下单」），且**当前上下文已满足目标功能**的必填条件时：

1. 弹出 **切换功能卡**（`card-cross-skill`），展示已具备的上下文摘要；  
2. **使用当前信息**：按携带规则进入目标功能（如方案→报价带入客户与方案/选品；报价→下单带入客户与报价单）；  
3. **重新走流程**：清空携带，从目标功能首步开始。

不满足目标功能条件时，按目标功能正常缺槽引导，不弹此卡。

| 当前功能 | 输入指向 | 上下文齐备示例 | 带入结果 |
|----------|----------|----------------|----------|
| 方案速配 | 产品报价 | 客户 + 已保存方案 / 方案选品已选 | 按方案报价或报价选品卡 |
| 产品报价 | 确认下单 | 客户 + 本客户报价单 / 已填价明细 | 选择报价单或下单确认 |
| 产品报价 | 方案速配 | 客户 + 已有方案可参考 | 进入方案速配 |

### 2.4 缺槽引导 + 定位表单（核心）

缺槽时须同时满足：**界面**（本步业务卡/抽屉，与步骤摘要一致）+ **引导话术** + **定位界面**（打开填写的目标）+ **语音/输入示例**（可选）。意图钉缺槽表列顺序：**界面 | 缺槽字段 | 引导话术 | 定位界面 | 语音**。

- **验收结构**：意图钉 `chat-llm` 按功能分块，每块内 **1. 步骤摘要** 与 **2. 缺槽引导** 同级（见 `annotation-spec-data.js` → `extraHtml`）。  
- **全流程 / 随当前流程**：仅缺槽引导表（无独立步骤摘要；选客户 S1 写在各功能步骤摘要首行）。  
- **不重复维护**：五类下一步见 §2.3.1；活跃卡语音见 §2.5。  
- 话术与文档 04 同句；`guideMissingSlot`；气泡 **【待填写】** + 滚入定位界面。

#### 键名索引（实现对照，完整表在意图钉各功能「2. 缺槽引导」）

| 功能 | 键名示例 |
|------|----------|
| 全流程 | `customer`、`intentNeedFeature` |
| 方案速配 | `planDemand`、`planPickProducts`、`planCartEmpty`（空→选品）、`planTemplate`；购物车卡槽位：购买数量、规格 |
| 产品报价 | `quoteSource`、`quoteNoScheme`、`quoteSchemePick`、`quotePickProducts`、`quoteLinePrice`、`quoteTemplate` |
| 确认下单 | `orderSource`、`orderNoQuotes`、`orderQuotePick`、`orderPickProducts`、`orderLinePrice`、`orderDetail` |
| 随当前流程 | `productMatchFail` |

### 2.5 活跃卡语音

选品卡→筛选/选品；选择方案·报价单卡→「第 N 条」或名称编号；逐项报价→行级改价/数量。与各功能「2. 缺槽引导」语音列一致，不另表维护。

### 2.6 无匹配兜底

| 情况 | 下一步类型 |
|------|------------|
| 完全无功能词 | 能力说明 |
| 仅客户名 | 缺槽引导（`intentNeedFeature`） |
| 像某功能但槽位不足 | 缺槽引导（首缺槽 `guideMissingSlot`） |
| 方案/报价单/产品名称对不上 | 缺槽引导或消歧（全量选择卡） |
| 流程冲突 / 点失效按钮 | 能力说明 |

### 2.7 实现索引

`guideMissingSlot` / `tryGuideAfterIntentFail` / `tryIntent` / `parse*Attributes`（`skills.js`）。

## 四、`tryIntent` 优先级（简表）
