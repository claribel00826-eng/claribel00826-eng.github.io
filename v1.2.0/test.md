# 标注文档 02：方案速配 · 产品报价 · 确认下单

> **版本**：1.2.0 版 · **入口**：欢迎区/技能条/意图话术  
> **前置**：多数步骤须已选**当前客户**（顶栏展示）；缺客户时出选客户引导卡。  
> **上下文**：会话内按客户保存方案、报价单、待确认报价、待确认订单等（按客户校验）。

---

## 一、主链路与解耦

```
选客户
  → 方案速配：选品改规格 → 购物车改规格 → 方案模板 → 方案卡（无价格）
  → 产品报价：按方案 / 直选品 → 逐项报价 → 报价单模板 → 报价单卡
  → 确认下单：按报价单 / 直选品（须逐项报价）→ 下单确认抽屉 → 待排产成功卡
```

| 原则 | 说明 |
|------|------|
| 方案无价格 | 方案卡、选品列表**不展示单价**；金额仅在报价阶段 |
| 报价≠下单 | 生成报价单**不**自动建单；须报价单卡「生成订单」或技能「确认下单」 |
| 一句话捷径 | 描述性话术可跳过中间卡片（详见文档 **03** §三、§四） |
| 历史页面 | 弹出新助手页后，上一页按钮失效（文档 **03** 第二节） |

## 二、方案速配

### 2.1 入口与引导

| 入口 | 行为 |
|------|------|
| 欢迎区「方案速配」/ 技能条 | 发话术「配个方案」→ 引导句 + **选品卡** |
| 待跟进「做方案」 | 切换客户后进入方案速配 |
| 一句话需求 | 直接 **方案卡**（03 §三） |

**引导句内容**：说明可一句话描述需求，或走选品 → 购物车 → 模板；强调方案不含价格。

**新客户分步**：须先进入 **需求描述卡**（卡片内多行文字 + 确认；**语音仅底部输入区**，卡片内不设按住说话），确认后再出选品卡；推荐区用**本次需求文本**匹配（不再仅用档案 `demandHint`）。老客户仍直接进入选品卡。

### 2.2 方案选品卡

#### 内容

1. 标题「选品 · 改规格」。
2. **推荐区**（列表上部，老客户与新客户规则不同）  
   - **老客户**（定义）：本企业下曾有过订单，且档案**责任人=当前登录业务员**。推荐区列出该客户历史订单里买过的产品，按下单日期从新到旧、去重后最多 10 条，标签「历史订单」。
   - **新客户**（定义）：跟进状态=**待跟进**，且**无责任人**。推荐区**两步**：① 按客户档案**客户性质**（`category`）与产品**产品类型**（`productType`）筛候选；② 用档案**需求描述关键词**（`demandHint`）与候选产品**存货描述**算匹配度，**超过 80%** 的前 10 条进推荐区，标签「匹配 xx%」。
3. **筛选**：搜索框 +「筛选」按钮；输入的字在**品名、规格、存货描述**里查找，**同时**过滤推荐区与更多产品。
4. **更多产品**（推荐区未展示的关联存货，见下「查询逻辑」）  
   - 标签「全部」；**首屏 5 条**，列表内下滑懒加载每页 5 条（03 §五）。  
5. 每行：品名、规格、匹配标签；**规格下拉始终可见**（未勾选也可改规格）。
6. 新客户且已录入本次需求时：卡内展示「当前需求：…」+ 按钮「修改需求」（购物车、预览卡无此按钮）。
7. 底部：「加入购物车」；语音提示文案。

#### 查询逻辑

**关联查询总则**

| 客户类型 | 推荐区怎么挑 | 更多产品怎么挑 |
|----------|--------------|----------------|
| 老客户 | 该客户历史订单里的产品，按日期从新到旧，最多 10 条 | 全库去掉推荐区已出现的；搜索框有字则品名/规格/存货描述包含才显示；无字则余量全显示；按品名排序 |
| 新客户 | ① 客户性质×产品类型筛候选 → ② 本次需求与「存货描述/品名」相似度 **>80%**，取前 10 条 | 同一候选池去掉推荐区已出现的其余产品（按相似度排序）；有筛选词时仅显示品名/规格/描述命中项 |

**新客户第一步（类型）**  
客户档案 `category`（客户性质）对应可售 `productType` 列表，演示见 `DemoData.categoryProductTypes`。

**新客户第二步（描述）**  
在候选池内，用 `demandHint` 与 `inventoryDesc` 算相似度；例如「伺服电机 密封件」拆词后命中一半，则 50%。

**推荐区**

- 老客户：历史订单产品，最多 10 条。  
- 新客户：类型筛选后的候选里，描述相似度超过 80% 的最多 10 条。

**更多产品**

- **老客户**：全库减去推荐区已展示。  
- **新客户**：类型候选池减去推荐区已展示。  
- 搜索框、排序规则见上表。  
- 分页：`moreVisible` 初值 5，筛选后重置为 5，触底每次 +5。

**上下文（`state.ctx.plan`，演示写入 `localStorage`）**

- 保存：已选品、规格（SKU）、数量、筛选词、更多区可见条数、已选方案模板。  
- **不失效**：用户仅发消息尚未弹新页、同客户再次进入方案速配、懒加载刷新选品卡、已生成方案卡之后。  
- **整包替换或清空**：顶栏换客户且 `customerId` 变化（`ensurePlan` 新建空对象）、演示「重置页面」、`ctx` 清空；切换企业后若当前客户不属于该企业，需重选客户后再进方案速配。  
- 默认规格来自演示主数据。

#### 交互逻辑

| 操作 | 结果 |
|------|------|
| 点行勾选 | 勾选/取消已选品 |
| 改规格下拉 | 更新该行规格 |
| 筛选 / 语音「筛选 xxx」 | 更新筛选词，重置更多区分页，刷新卡片 |
| 语音「选品 伺服电机」 | 按关键词勾选并刷新 |
| 加入购物车 | 将选品卡勾选行写入购物车 → 出 **方案购物车卡**（此后改购买数量） |
| 语音「加购」 | 同左 |
| 修改需求（仅新客户、仅选品卡） | 点按钮或主对话「修改需求 / 改需求 / 重新描述需求 / 变更需求」→ **修改采购需求**提示卡；带正文（≥2 字）可直接覆盖需求并重出选品卡；**清空已勾选**并重算推荐区 |

#### 修改需求（新客户）

| 项目 | 说明 |
|------|------|
| 入口 | 选品卡「修改需求」；主对话口令见上表 |
| 限制 | 仅新客户；方案购物车、预览、模板、方案卡不可改需求 |
| 处置 | 覆盖「本次需求」文本 → 推荐区按新需求重算；已选品、筛选词、更多区分页归零 |

### 2.3 方案购物车卡

#### 内容

- **加购后**展示已勾选行：品名、规格下拉、**购买数量**输入（本步主要确认数量与规格）。
- 「返回选品」补勾产品；「生成方案」须先选模板。

#### 交互

- **购买数量与规格**在此卡调整（选品卡勾选后写入购物车，不在选品卡做最终数量确认）。  
- **生成方案** → 打开 **方案模板抽屉**（非直接出方案卡）。  
- 语音「生成方案」→ 打开模板抽屉。

### 2.4 方案模板抽屉

#### 内容

- 单选一种方案模板（序号 + 名称 + 描述）。

#### 交互

- 点选或底部 **语音/键盘**：「第 1 个」「2」「标准技术方案」等选行；「保存方案」提交。  
- 未选模板时轻提示。  
- 确认后写入当前客户方案 → 关闭抽屉 → 会话插入 **方案卡**。  
- 方案行：品名·规格·数量；**无价格字段展示**。  
- 抽屉所选为**版式模板**（排版用）；**方案名称**在保存时按 **§2.5** 规则生成。

### 2.5 方案卡

#### 内容

- 方案编号、客户、**方案名称**、版式模板名、行明细（品名 · 规格 · 数量）。  
- 按钮：「预览 PDF」「去报价」。  
- **方案名称**（保存时自动生成，非版式模板名）：**客户名 + 产品 + 日期**，三部分直接拼接；产品取已勾选品名（1 项用该名，多项用「第一件+等」）；日期为保存日 `yyyyMMdd`。示例：`深圳创源伺服电机等20260521`。版式模板名来自方案模板抽屉，与方案名称不同。

#### 查询

- 当前客户方案：编号、客户、方案名称、版式模板、行明细  
- 主对话可按方案名称或方案编号检索

#### 交互

- **预览 PDF**：全屏打开**技术方案**预览页（HTML 模拟）；表头序/品名/规格/数量/单位；页脚合计数量，**无单价与金额**。  
- **去报价**：无方案时提示先完成方案速配；有方案 → 载入方案行 → **逐项报价抽屉**（或一句话直接报价单，见文档 03 第四节）。  
- 无「生成订单」按钮（须先报价）。

## 三、产品报价

**新客户（报价直选）**：与方案速配相同，须先 **需求描述卡**（`card-quote-demand`），确认后再出 **报价选品卡**；推荐区用本次需求文本匹配。若同客户已在方案流程录入需求，新建报价可沿用该文本。老客户直进选品。

### 3.1 报价来源卡

#### 内容

- 「按方案报价」「直接选品报价」；有方案时提示方案数量。

#### 查询

- 本客户已保存方案列表（会话内可多个）。

#### 交互

| 按钮 | 行为 |
|------|------|
| 按方案报价 | **仅 1 个方案**：载入该行 → **逐项报价抽屉**；**多个方案**：须先 **选择方案卡**（展示**方案名称**）→ 再逐项报价或一句话出报价单卡 |
| 直选 | **报价选品卡** |

方案卡「去报价」带方案编号，多方案时**直接按该卡方案**报价，不再弹出选择卡。

**一句话报价**：可跳过本卡（见文档 03 第四节）。

### 3.2 报价选品卡

- 推荐/筛选/规格规则同方案选品；新客户须先完成 §三开头需求步骤。  
- 已录需求时：卡内「当前需求」+「修改需求」（规则同方案选品卡 §2.2）。  
- 「下一步：逐项报价」→ 生成待确认报价行（默认最新售价）→ **报价选品确认卡** 或进入逐项抽屉（直选可走确认卡再模板）。

### 3.3 报价选品确认卡

#### 内容

- 每行：规格、数量、最新/最低售价、**本单报价**输入。  
- 报价合计；直选可勾选「保存为方案」。

#### 交互

- 填价 → 「下一步：选择模板」→ **报价单模板抽屉**。  
- 行内改价/数量同步待确认报价明细。

### 3.4 逐项报价抽屉与语音条

#### 内容

- 按方案/直选/下单直选载入的行：规格、数量、最新/最低售价、本单报价。  
- 低于最低价行高亮警告。  
- 下单模式标题「逐项报价（下单）」；按钮「生成订单」。  
- **底部语音条**：按住说话 / 键盘；识别**不进对话区**。

#### 查询

- 待确认报价：行明细、是否下单填价模式、是否保存为方案、来源类型、关联方案编号

#### 交互（语音）

- 填价：「第一项报价 3680」「伺服电机 报价 4200」。  
- 改数量/规格。  
- 报价模式：「选择模板」→ 模板抽屉。  
- 下单填价模式：「生成订单」→ **下单确认抽屉**。

### 3.5 报价单模板抽屉

- 单选一种报价单版式 → 确认 → **报价单卡**。

### 3.6 报价单卡

#### 内容

- 报价号、客户、模板名、**报价金额**。  
- 「看 PDF」「生成订单」。

#### 查询

- 当前客户报价单

#### 交互

- **看 PDF**：全屏打开**销售报价单**预览页；表头序/品名/规格/数量/单位/单价/金额；页脚**含税合计**为各行金额之和。  
- **生成订单** → **下单确认抽屉**（沿用报价行与金额）。  
- **不**自动创建订单记录直至下单抽屉确认。

### 3.7 方案 / 报价单 PDF 预览层

- 全屏层（`modal-pdf`）；顶栏标题为「方案预览」或「报价单预览」，「返回」关闭。  
- 正文为白底文档样式（演示 HTML，非真实 PDF 下载）。  
- 数据与当前会话中方案对象、报价单对象一致。

## 四、确认下单

### 4.1 下单来源卡

| 来源 | 条件 | 下一步 |
|------|------|--------|
| 按报价单 | 本客户已有报价单 | **仅 1 个**：直接 **下单确认抽屉**；**多个**：先 **选择报价单卡** |
| 直接选品 | 已选客户 | **订单选品卡** |

报价单卡「生成订单」带报价单编号，多报价单时**直接按该卡报价单**下单，不再弹出选择卡。

### 4.2 订单选品卡

- 推荐/筛选/规格/数量同报价直选。  
- 「下一步：逐项报价」→ 下单填价模式 → **逐项报价抽屉**。  
- 可选「保存为方案」勾选。  
- 直选下单**必须**先逐项填价（无「按报价单跳过」除非已有报价单）。

### 4.3 下单确认抽屉

#### 内容

- 客户、来源摘要（报价单号+金额 **或** 直选行数+合计）。  
- 可选展示交期说明（本版主链路不强制交期）。

#### 交互

- **确认下单** → 写入演示订单库（状态**待排产**，含行明细、业务员）→ 关闭抽屉 → **订单成功卡**。

### 4.4 订单成功卡

- 订单号、客户、状态、**订单明细**（品名、规格、数量与单位）、合计金额；只读。
