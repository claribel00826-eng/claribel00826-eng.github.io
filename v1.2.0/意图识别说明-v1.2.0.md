### 交互模式定义

| 模式 | 定义 |
|------|------|
| 完整需求 | **跟进 / 方案 / 报价 / 订单** 四功能之一：一次说齐该功能**全部必填槽位**，直达最终页（方案→方案卡，报价→报价单卡，订单→**下单确认卡**；不含订单成功页） |
| 部分需求 | 一句话仅含部分槽位或模糊意图，系统识别缺失项并引导至最相关子步骤 |
| 表单填充 | 在含表单字段的卡片上通过文字填写字段，填充后进入完成或下一步 |
| 模拟点击 | 口语化操作（如「帮我新建一个方案」）或念按钮/列表项（如「第1条」）；多步用 ` -> ` 连接 |

已省略 **功能意图**（与用户输入重复）。

## 特别说明 · 口令识别与推荐商品

本清单以**口令/话术识别**为主要视角。与 UI 卡片点击不同：

| 维度 | UI 卡片点击 | 口令/话术识别 |
|------|-------------|---------------|
| **推荐商品** | 新客户须先填需求；老客户可「跳过」并按历史订单推荐 | **新老客户逻辑相同**：均按需求文本对产品全库做模糊匹配，按匹配分排序取推荐区 |
| **缺需求时** | 新客户阻断推荐；老客户可走历史 Top10 | **新老客户同引导**：无话术品名/需求时，均缺「需求描述/品名」并引导至需求引导卡 |
| **更多产品区** | 老客户无筛选时按品名升序；新客户按匹配分降序 | **口令路径统一**：有需求文本时均按匹配分降序 |

> **用户输入**列只放用户口语（如「帮我切换客户到某某」），不放「点选」「（已选客户）」等说明；备注写在**解析槽位**。

> **初始化页面**仅登记导航与选客（多为 `模拟点击`）；从对话页一句进入方案/报价/下单子流程的行，**所属功能**归入对应业务能力。
