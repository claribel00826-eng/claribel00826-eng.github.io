## 本版目标

在 v1.1.0 对话壳基础上，交付 **方案速配 → 产品报价 → 确认下单** 主链路（纯前端模拟），与待跟进/写跟进并列可用。

## 范围（v1.2.0 新增 / 强化）

| 功能 | 用户入口 | 前置条件 | 产出 |
|------|----------|----------|------|
| **方案速配** | 欢迎功能格 / 技能条 / 话术「配个方案」 | 已选客户 | 推荐选品&规格 → 购物车&规格 → 选模板 → 方案卡（**不含价格**） |
| **产品报价** | 技能条 / 话术「报价」 | 已选客户 | 按方案或直选规格 → 报价设置 → **报价单模板** → 报价单卡 |
| **确认下单** | 报价单卡「生成订单」/ 技能条「确认下单」 | 已选客户 | 按报价单或直选规格（可选保存方案）→ **原订单确认抽屉** → 待排产 |

## 主链路（对话内）

```
选客户 → 方案速配：选品&改规格 → 购物车&改规格 → 选模板生成方案（无价格）
    → 产品报价（折扣）→ 生成报价单
    → 确认下单（按报价单或直选）→ 订单号
```

- **商品推荐**：老客户=历史订单 Top10；新客户=先客户性质×产品类型，再需求描述关键词×存货描述相似度>80% 的 Top10。
- **操作**：表单 + 语音/文本指令（选品、加购、生成方案、筛选）；**一句话描述需求**可跳过逐步选品，直接生成方案卡或报价单卡（演示：按存货描述/品名匹配 + 默认模板）。
- **产品槽**：选品 + 规格（含 SKU 标签）；方案正文不含价格，金额仅在报价阶段展示。
- **交期评审**：本版不纳入验收范围（技能仍可见，可 toast 或保留演示入口，主链路不依赖交期）。

## 意图与槽位（增量）

| 槽位 | 方案速配 | 报价 | 下单 |
|------|----------|------|------|
| 功能 | 必认 | 必认 | 必认 |
| 客户 | 必填 | 必填 | 必填 |
| 产品 | 对话内多选 + 数量 | 直选时多选 + 规格 | 直选时多选 + 规格 |

缺方案时「按方案报价」→ 提示先完成方案速配；无报价时仍可「直接选品」下单。

## 页面与交互要点

### 方案速配

1. 推荐列表（老/新客规则）+ 筛选 + 选品行内改规格
2. 「加入购物车」→ 购物车改规格/数量 →「生成方案」→ **选择模板** → 保存
3. 方案卡：模板名、明细（品名·规格·数量），**无价格**；按钮「预览 PDF」「去报价」

### 产品报价

1. **来源**：按当前方案 / 直接选品+规格（可选「保存为方案」）
2. **报价设置**：明细、折扣 0–30%
3. **报价单模板**：必选后生成报价
4. **生成后**：报价单卡（看 PDF、**生成订单**）；**不**自动建单

### 确认下单

1. **来源**：按报价单（须本客户已有报价）/ 直接选品+规格（可选保存为方案）
2. 直选：选品 → 购物车 →「生成订单」→ 写入 `ctx.orderPending`
3. **确认表单**（`sheet-order` 结构不变）：客户、来源摘要（报价单号+金额或行数+合计）；有交期则展示
4. 确认 → 订单成功卡（**待排产**）；写入 `DemoData.orders`（含行：存货编码/名称/规格/单位/数量、业务员）

## 数据（演示）

- 方案/报价/下单上下文保存在 `state.ctx`（`scheme`、`quote`、`orderDraft`、`orderPending`），按当前客户校验。
- 下单写入 `DemoData.orders` 首条，刷新后可在「订单进度」查看（v1.1.0 能力）。
