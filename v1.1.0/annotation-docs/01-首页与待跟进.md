# 标注文档 01：对话首页与待跟进

> **范围**：登录后「对话页」首屏 + 待跟进全链路。  
> **原型**：演示数据 + 应用脚本；正式环境以「查询逻辑」为准。  
> **落页**：确认后按第七节映射表写入网页。

---

## 一、整体结构

### 1.1 四大类

| 大类 | 包含模块 | 常驻 |
|------|----------|------|
| 一、对话壳 | 顶栏、当前客户条、消息区、技能条、输入区 | 是 |
| 二、首屏默认区 | 欢迎区、模板消息卡、待跟进摘要卡、最近访问 | 首屏写入消息流 |
| 三、待跟进业务流 | 列表卡、企业详情卡、下一步引导卡、写跟进抽屉 | 按操作出现 |
| 四、大模型能力 | 意图分配、场景回复 | 贯穿输入与技能条 |

### 1.2 首屏默认可见（自上而下）

顶栏 → 当前客户条 → 欢迎区 →（模板入口时）模板消息卡 → 待跟进摘要卡 → 最近访问 → 技能条 → 输入区。

### 1.3 待跟进子模块

待跟进摘要卡、待跟进列表卡、企业详情卡、下一步引导卡、写跟进抽屉。  
列表为**今日待跟进企业列表**（新客户与老客户合并，无分 tab），行内徽标区分归属；**待跟进企业数量**为列表中的企业数。

### 1.4 触发后出现

待跟进列表卡、企业详情卡、下一步引导卡、写跟进抽屉、切换客户弹窗、切换企业抽屉。

## 二、对话壳

### 模块 2.1：顶栏

1. 标题：「销售助手 · Pro」（仅智能体名，无企业入口）

### 模块 2.2：当前客户条

1. 未选显示「请选择客户」；已选显示客户全称（可截断）
2. 已选时展示：客户性质 + 新/老客户徽标（规则见「新客户/老客户待跟进查询」）
   - **查询逻辑**：来往单位主数据；弹窗 Tab 为**客户 / 供应商 / 供应商/客户**，支持名称/编码搜索。当前客户条上的**客户性质**（如终端客户）为另一字段。
   - **交互逻辑**：点击 → 切换客户弹窗 → 选中 → 轻提示「已切换至某某」→ 刷新条栏；不打开企业详情卡。
3. **清空记录**（独立标注 `btn-clear-customer-chat`）：确认后删除消息区**全部**记录（含欢迎区、待跟进摘要、最近访问及后续对话），并清空本地缓存；同日不会自动再发每日首屏，次日首次进入再发。

### 模块 2.3：消息区 / 技能条 / 输入区

- **消息区**：智能助手气泡、用户气泡、卡片；纵向滚动，新消息滚到底部。
- **技能条、输入区**：见文档 03。

## 三、首屏默认区

> 首屏内容由「初始化欢迎」写入消息区。

**消息区 · 会话与刷新规则**

| 场景 | 规则 |
|------|------|
| 每日首次进入 | 每个自然日、每企业**第一次**打开对话页：发送欢迎区 + 待跟进摘要 + 最近访问（有历史则插到顶部） |
| 刷新页面 | 同日再次进入：从本地恢复完整消息流；单企业最多 200 条 |
| 无历史 | 写入首屏三块（欢迎、待跟进摘要、最近访问） |
| 重新登录 | 清空会话与每日标记，重新首屏 |
| 每条新消息 | 自动保存当前企业会话；已选客户时消息带客户标识 |
| 清空记录 | 删除消息区**全部**内容（含系统首屏三块）及本地缓存；不自动重发首屏 |
| 模板入口 `?tpl=followup` | 页面左侧微信区展示服务号模板消息卡（不在 H5 对话流内）；点击按钮进入 H5 并拉待跟进列表 |

### 模块 3.1：欢迎区

1. 主文案：「你好，我是销售助手。可点下方待跟进查看企业，或使用底部技能完成方案、报价、交期、下单等操作。」
2. 副文案：「支持：待跟进、方案速配、报价、交期、订单、复制订单、变更、查进度、客服工单。」
   - **查询逻辑**：静态文案。
   - **交互逻辑**：首屏第一条智能助手内容；无点击。

### 模块 3.2：待跟进摘要卡

1. 标题「今日待跟进」；展示**待跟进企业数量**（如「3 家企业」）；引导「点击查看列表」
2. 数量：符合新客户或老客户待跟进条件的往来单位**合计**
   - **查询逻辑**：调用今日待跟进查询，得到**今日待跟进企业列表**与**待跟进企业数量**；排序与条数见下文 6.4；与初始化欢迎同时拉数。
   - **交互逻辑**：点击摘要 → 用户消息「今日待跟进」→ 约 300 毫秒后回复待跟进列表卡。数量为 0 时显示「0 家企业」，可点击，列表为空。

### 模块 3.4：模板消息卡（服务号 · 待跟进）

> 微信服务号推送的模板消息样式由微信侧定义；演示页在 H5 外侧（左侧微信区）用本卡还原推送语义与跳转。

1. 渠道标签「服务号消息」；标题「今日待跟进提醒」；正文含待跟进企业数量 N；按钮「查看待跟进列表」
   - **查询逻辑**：模板文案由服务号配置；N 与今日待跟进查询接口一致。深链 `tpl=followup` 打开对话页并触发本卡。
   - **交互逻辑**：用户在服务号点击模板消息 → 进入演示页（左侧本卡 + 右侧 H5）→ 点击按钮 → H5 内用户消息「今日待跟进」→ 回复待跟进列表卡（与摘要卡、技能「待跟进」一致）。

### 模块 3.3：最近访问

1. 标题「最近访问」；项为「客户名 · 时间标签」，当前企业下最多 3 条（原型）
   - **查询逻辑**：正式环境为最近访问记录，按时间倒序取前三。
   - **交互逻辑**：紧接摘要卡后插入。点击某项 → 切换当前客户 + 底部对应 Skill 高亮 → 模拟发送话术 → 执行该功能（待跟进出列表卡，报价/方案等走 Skill 流程）。

## 四、待跟进业务流

### 模块 4.1：待跟进列表卡

1. 标题「今日待跟进 · X 家」（X 为待跟进企业数量）；行：企业名称 + 新/老客户徽标
   - **查询逻辑**：与摘要卡共用同一份**今日待跟进企业列表**与**待跟进企业数量**；排序与条数见 6.4。
   - **交互逻辑**：点击摘要、话术/技能「待跟进」、意图待跟进列表时在消息流**追加**列表卡（可多次）。点击行 → 设当前客户（静默）→ 详情卡 + 下一步引导卡。

### 模块 4.2：企业详情卡

1. 标题「企业详情」+ 新/老客户徽标
2. 字段：编码、名称、性质、客户性质、结算客户、客户等级、联系人地址（姓名·电话·地址）
   - **查询逻辑**：按客户标识取主数据/详情接口。
   - **交互逻辑**：从列表或最近访问选企业后追加；只读。

### 模块 4.3：下一步引导卡

1. 标题「接下来您想做什么？」；按钮：写跟进 | 做方案 | 稍后再说
   - **交互逻辑**：紧接详情卡追加。写跟进 → 模块 4.4；做方案 → 文档 03；稍后再说 → 智能助手固定回复。

### 模块 4.4：写跟进抽屉

1. 字段：联系人*、联系方式*、发货地址、跟进信息*、跟进时间*、跟进状态（跟进中 / 跟进完成）
2. 表单「跟进状态」用于本次提交；新客户待跟进条件中的「待跟进」指**历史最新一条跟进记录**的状态，二者不同。
   - **查询逻辑**：正式环境提交跟进接口，关联客户、企业、跟进人；原型前端校验。
   - **交互逻辑**：有当前客户时打开，标题「写跟进 — {企业名}」，默认带入联系人信息。无当前客户时意图写跟进 → 提示先从待跟进选择。提交成功 → 关闭 → 用户消息「已提交跟进记录」→ 智能助手确认气泡。× 或点遮罩关闭。

## 六、新客户 / 老客户待跟进查询

> 用于待跟进摘要数量、待跟进列表行、新/老客户徽标。输出**今日待跟进企业列表**（合并，无 tab）与**待跟进企业数量**。

### 6.1 总体

- **范围**：当前登录用户、当前企业下往来单位。
- **合并**：老客户待跟进 ∪ 新客户待跟进 → **今日待跟进企业列表**；**待跟进企业数量** = 列表中的企业数；每家单位仅属一类。
- **判定顺序**：先判老客户；否则判新客户。两类均不满足则不进入列表。
- **列表行**：单位名称、新/老客户徽标。

### 6.2 老客户（须同时满足）

1. **分管关系**：往来单位分管人员 = 当前登录用户。
2. **超时未下单**：
   - 数据：主数据「销售提醒天数」+ 订单「最后一单日期」或「最近有效订单日期」（以企业主数据配置为准）。
   - 规则：当前日期 − 基准日期 > 销售提醒天数 → 超时未下单。
   - 无订单日期：按企业主数据约定是否视为超时（须配置明确，不写默认口径）。

归入老客户 → 进入今日待跟进企业列表，徽标「老客户」。

### 6.3 新客户（须同时满足）

1. **无分管**：分管人员为空，或主数据为公海/无负责人。
2. **跟进状态**：跟进记录按时间取最新一条，其「跟进状态」=「待跟进」（与写跟进表单「跟进中/完成」不同）。无跟进记录则不归入新客户。

归入新客户 → 进入今日待跟进企业列表，徽标「新客户」。

### 6.4 合并、排序、条数与刷新

1. 查老客户待跟进（6.2）→ 查新客户待跟进（6.3）→ 合并为今日待跟进企业列表 → 计数为待跟进企业数量。
2. **排序**（合并后统一排序，摘要与列表共用）：**最近更新时间倒序**（往来单位或跟进记录的最后更新时间，以接口字段为准）；同时间可按单位名称升序。
3. **条数**：一次返回列表**全量**；摘要卡数量与列表卡行数一致，首屏不截断。若企业配置单次返回上限，列表分页规则由接口约定。
4. 摘要卡与列表卡共用同一份列表与数量。
5. **拉数时机**：首屏、展开列表、切换企业、写跟进提交成功后（提交后是否刷新由接口约定）。
6. **接口失败**：轻提示；数量可显示上次结果或「—」。

### 6.5 展示相关交互

- 列表、详情卡、当前客户条展示新/老客户徽标。
- 不提供按新/老客户分 tab 筛选。
- 待跟进企业数量为 0：摘要显示「0 家企业」；展开列表为空。
- 选中行：写入客户槽，拉取详情。

### 6.6 原型

演示数据按企业取前 3 条；徽标用演示「客户类型」字段，不用于正式验收。
