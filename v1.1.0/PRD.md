## 1. 背景与调研结论

### 1.1 产品愿景

将原有制造业 SaaS 移动端改造为 **单页对话智能体**：外勤销售登录后仅在 `/chat` 完成客户跟进及后续方案/报价/交期/下单等能力；自然语言 + 语音为主，Skill 在对话内以卡片与轻表单（含底部抽屉）完成，无功能宫格、无多级路由跳转。

### 1.2 阶段 R 结论摘要

详见 [../shared/competitive-analysis.md](../shared/competitive-analysis.md)（跨版本维护）。

### 1.3 体验红线

- 面向销售人员的界面符合 **大众 H5 习惯**，不出现「标注模式」「设计稿开关」等产品化入口。
- 设计标注仅用于交付验收：默认隐藏，可通过 `?spec=1` 或快捷键打开侧边标注面板（不对最终用户宣传）。

## 2. Mission / Persona

### Mission

让外勤销售在拜访路上 **用说话的方式** 完成「看谁要跟 → 记跟进 →（后续）方案到下单」，减少在多个菜单间跳转的时间。

### Persona

| 角色 | 描述 |
|------|------|
| **主用户：外勤销售** | 25–45 岁，常用微信打开 H5；单手操作；需要快速看今日待跟进、记一笔跟进 |
| 次要（V2+） | 内勤跟单、客服（同一套壳，权限与 Skill 可见性可扩展） |

## 3. 版本规划

### V1 / MVP（当前原型范围）

| 能力 | 深度 |
|------|------|
| `/login` | 手机号/账号 + 密码（模拟），Token 写入 localStorage |
| `/chat` 对话壳 | 顶栏（智能体、企业切换、客户选择）；消息区；底部输入 |
| **首屏 /「首页」** | `/chat` 空态或首进：AI 欢迎语 + **快捷话术** 3 条 |
| **Skill 横滑条** | 输入框上方，参考豆包移动端，左右滑动选择 Skill |
| **F04 待跟进** | 列表卡片、客户详情、写跟进（内嵌 + 底部抽屉） |
| 全局上下文 | 当前企业、当前客户；切换后对话内系统提示 |
| 虚拟数据 | ≥2 企业、≥3 客户、待跟进列表 mockData |
| PDF 预览 | 通用预览层组件（占位/模拟 PDF，供后续方案卡复用） |
| 语音 | UI + 模拟「按住说话 → 填入输入框」（无真实 ASR） |

**不含（明确延后 V2+）**：方案速配、报价、交期、下单、复制、变更、进度、客服的完整业务闭环；服务号深链完整联调（可预留 URL 参数钩子）。

### V2+

- 主链路 Skill：方案 → 报价 → 交期 → 下单
- 订单复制 / 变更 / 进度 / 客服
- 服务号推送深链 `customer_id` 自动设客 + 主动话术
- 8 Skill 横滑条全部可点可用

### MVP 取舍理由

外勤最高频入口是 **「今天跟谁」**；先验证对话壳 + Skill 条 + 待跟进卡片交互，再叠主链路，降低原型返工风险。

## 5. 主功能定义（A2）

### F01 账号与会话

- **做**：无 Token → `/login`；成功 → `/chat`；Token 本地维护。
- **不做**：注册、找回密码、多因素认证。
- **完成标准**：刷新后仍进 `/chat`（有 Token）。

### F16 对话壳 + Skill 横滑条

- **做**：
  - 唯一业务页 `/chat`；
  - 输入框**上方**固定 **Skill 横滑条**（豆包移动端范式）：可左右滑动，选中项高亮；点击选中 Skill 后，在输入框插入提示文案或直接向 AI 发起该 Skill 默认意图；
  - MVP 仅 **「待跟进」** 可执行，其余 Skill 显示但点击 toast「即将开放」或置灰（产品择一，原型默认：可点 + toast）。
- **不做**：独立 Skill 全屏页、宫格首页。
- **依赖**：无。

### F04 待跟进

- **做**：
  - 触发：自然语言（「今天要跟哪些客户」「待跟进」）/ 快捷话术「今日待跟进」/ Skill 条选中「待跟进」；
  - 输出：**待跟进列表卡片**（客户名、最后下单、等级、条数 n）；
  - 卡片按钮：**查看详情** | **写跟进** | **做方案**（MVP：toast 即将开放）；
  - 写跟进：对话内嵌条或 **底部抽屉**（跟进内容 + 提交）；提交后 AI 回复「已记录跟进：XXX」；
  - 查看详情：客户摘要卡片（联系人、等级、最后下单等）。
- **不做**：真实写库、跨 Skill 改方案数据。
- **完成标准**：完整走通「列表 → 详情 → 写跟进 → 成功反馈」且全程不跳转路由。

### F15 意图（MVP 简化）

- **做**：规则 + 关键词模拟（非真实 LLM）；识别「待跟进」「写跟进」「切换客户」等；有当前客户时自动带入。
- **不做**：复杂多轮编排；无槽位时 **最多 1 句澄清**（如「要给哪位客户写跟进？」）。

### 全局：企业与客户

- **企业切换**：顶栏切换 ≥2 家 mock 企业 → 对话内提示「已切换企业：XX」→ 待跟进列表数据切换。
- **客户选择**：顶栏右侧按钮 → **底部抽屉**搜索选择 → 对话内提示「已切换客户：XX」。

## 6. 页面清单与跳转（A4）

> **原则**：无 `/home` 宫格；「首页」= `/chat` 首屏态。

| 路由 | 类型 | 承载功能 |
|------|------|----------|
| `/login` | 独立页（可 hash `#login` 与 chat 同 HTML） | F01 |
| `/chat` | **唯一业务页** | F16 壳、F04、F15、企业/客户、快捷话术、Skill 条 |

### 跳转逻辑

```
打开 H5
  ├─ 无 Token → /login
  └─ 有 Token → /chat（首屏欢迎态）
/login 成功 → /chat（replace，不进入 /home）
```

### `/chat` 布局结构（自上而下）

```
┌─────────────────────────────────────┐
│ 顶栏：智能体名称 | 企业▼ | 客户👤    │
├─────────────────────────────────────┤
│                                     │
│  消息区（气泡 + 业务卡片 + 系统提示）   │
│                                     │
├─────────────────────────────────────┤
│  快捷话术（3 条，首屏/空态时显眼）      │  ← 欢迎语 | 今日待跟进 | 最近访问
├─────────────────────────────────────┤
│  Skill 横滑条（豆包式，可左右滑）       │  ← 待跟进|方案|报价|…（MVP 仅待跟进可用）
├─────────────────────────────────────┤
│  输入区：🎤 语音（默认）| 文字 | 发送   │
└─────────────────────────────────────┘
```

### 快捷话术 vs Skill 条（已确认）

| 区域 | 内容 | 交互 |
|------|------|------|
| **快捷话术**（3 条） | ① **欢迎语**（如「你好，我是销售助手」— 点击后 AI 回复能力说明）<br>② **今日待跟进**（等同触发 F04）<br>③ **最近访问**（列出最近 1–3 个客户快捷卡，点击设当前客户并提示） | 点击即发送/触发，出现在 **Skill 条上方** 或首屏消息区下方（实现时以不挡消息为准，优先：**紧贴 Skill 条上方**） |
| **Skill 横滑条** | MVP 展示 8 个 Skill 名称；仅「待跟进」可用 | 左右滑动；选中态描边/底色；点击发起对应 Skill |

## 7. 复杂链路（A5）— 纯前端实现思路

### 7.1 待跟进主链路

```
用户：点击「今日待跟进」| 选 Skill「待跟进」| 说「今天要跟哪些客户」
  → intent: follow_up_list
  → 渲染 FollowUpListCard（mockData.followUps 按 enterpriseId 过滤）
用户：点击「查看详情」
  → 渲染 CustomerDetailCard（当前 row 客户）
用户：点击「写跟进」
  → 若 !currentCustomerId → AI 澄清 + 打开客户抽屉
  → 否则 openBottomSheet(FollowUpForm)
用户：提交
  → push 用户消息（跟进正文）
  → push AI 消息「已记录跟进：{text}」+ 更新 mock 状态（可选）
```

### 7.2 最近访问

```
mockData.recentCustomers: [{ id, name, visitedAt }, ...] 最多 3 条
点击某条 → setCurrentCustomer → 系统消息「已切换客户：XX」
```

### 7.3 企业切换

```
顶栏企业 Selector → 切换 enterpriseId
  → 系统消息「已切换企业：XX」
  → 清空或保留客户（规则：切换企业后若客户不属于该企业则清空当前客户并提示）
```

### 7.4 语音（模拟）

```
按住 mic → 显示录音态 → 松开 → 将预设或随机一句中文填入输入框（editable）→ 用户可发送
```

### 7.5 PDF 预览（MVP 占位）

```
任意「预览 PDF」按钮 → openPdfModal(url | blob | static-sample.pdf)
全屏层：顶栏关闭 + iframe/img 模拟
供 V2 方案卡/报价卡复用
```

### 7.6 标注系统（交付用，非产品功能）

```
默认 annotationPanel hidden
URL ?spec=1 → 显示右侧标注面板；点击带 data-spec-id 的模块显示 token
按模块标注：顶栏、消息气泡、待跟进卡片、抽屉、Skill 条、输入区
```

### 7.7 mockData 结构（草案）

```javascript
const mockData = {
  enterprises: [{ id, name }],
  customers: [{ id, enterpriseId, name, level, lastOrderAt, contact }],
  followUps: [{ customerId, customerName, level, lastOrderAt }],
  recentCustomers: [{ customerId, name, visitedAt }],
  messages: [] // 运行时追加
};
```

## 8. 设计规范与标注确认清单（A6）

> 请用户逐项打勾确认；全部确认后进入阶段 B1。

### 8.1 视觉设计规范（提议默认值，源自 UI Design Standard V6.1）

#### 色彩系统

- [ ] 主色：`#4F46E5`（Indigo，按钮/选中 Skill/链接）
- [ ] 辅助色：`#71717A`（Zinc-500，次要文字、边框）
- [ ] 页面背景：`#FAFAFA`（Zinc-50）
- [ ] 卡片背景：`#FFFFFF`
- [ ] 文字主色：`#18181B`（Off-Black）
- [ ] 文字辅助：`#71717A`
- [ ] 错误/警告：`#DC2626` / `#D97706`

#### 字体系统

- [ ] 标题字体：`Outfit`（Google Fonts），字距 -0.04em
- [ ] 正文字体：`Geist` 或系统 `-apple-system` 回退
- [ ] 字号：标题 18px / 卡片标题 16px / 正文 14px / 辅助 12px
- [ ] 字重：标题 600，正文 400

#### 间距与圆角

- [ ] 间距基准：4px 倍数（8/12/16）
- [ ] 圆角：按钮、卡片、输入框 **8px**（≤12px）
- [ ] 卡片阴影：`0 2px 8px rgba(0,0,0,0.08)`

#### 布局规则

- [ ] 移动端优先，最小宽度 320px，安全区适配 iOS bottom
- [ ] Skill 横滑条高度约 40–44px，chip 间距 8px
- [ ] 顶栏高度 48px（含状态栏时用 `env(safe-area-inset-top)`）

#### 组件风格（YonClaw + 豆包融合）

- [ ] 用户气泡：浅灰底 `#F4F4F5`；AI/系统：白底或浅紫底 `#EEF2FF`
- [ ] 业务卡片：白底 + 8px 圆角 + 轻阴影，操作区主按钮实心、次按钮描边
- [ ] M3 五态：Hover/Focus/Pressed/Disabled 对可点击元素生效

### 8.2 标注内容确认清单

- [ ] 标注维度：尺寸、margin/padding、字体、背景、圆角、阴影、hover（按模块汇总，非逐像素）
- [ ] 标注模块列表：顶栏 | 消息区 | 快捷话术 | Skill 横滑条 | 输入区 | 待跟进卡片 | 客户详情卡 | 跟进抽屉 | PDF 预览层
- [ ] 标注展示方式：**右侧固定面板** + 点击模块高亮（**默认关闭**，`?spec=1` 开启）

### 8.3 技术栈（实现阶段）

- [ ] 单 HTML 文件（或多页 hash：`#login` / `#chat`）
- [ ] 纯 CSS + 原生 JS，`mockData` 驱动
- [ ] 不使用标注相关文案作为界面可见功能
