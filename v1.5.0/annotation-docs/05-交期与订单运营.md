## 一、模块链路表

| 大类 | 触发 | 链路（中文） | 标注标识 |
|------|------|--------------|----------|
| 交期评审 | 技能 / 话术 | 入口 → 选择来源 → 选单/需求/选品 → 表单 → 结果 | `card-delivery-entry` → `card-delivery-source` → … → `sheet-delivery` → `card-delivery` |
| 复制订单 | 技能「复制订单」 | 需求筛选 → 选择历史单 → 明细确认 → 下单确认（02） | `card-copy-demand` → `card-order-pick` → `card-order-copy` → `sheet-order` |
| 订单变更 | 技能「订单变更」 | 选择历史单 → 确认变更 → 变更已提交 | `card-order-pick` → `card-change-confirm` → `card-change-success` |
| 订单进度 | 技能 / 话术 | 需求筛选 → 选择订单 → 进度详情 | `card-progress-demand` → `card-order-pick` → `card-order-progress-detail` |

交期选单中间卡：`card-delivery-scheme-pick`、`card-delivery-quote-pick`、`card-delivery-order-pick`、`card-delivery-demand`；自选选品复用 `card-order-pick`（交期模式）。

## 三、三块摘要

| 中文模块名 | 一、展示与口径 | 二、范围 | 三、交互 |
|------------|----------------|----------|----------|
| 交期评审 · 入口 | 客户名、类型标签、评估交期按钮 | 顶栏已选客户 | → 选择来源 |
| 交期评审 · 选择来源 | 四来源分组；无数据置灰与数量副文案 | — | → 选单/需求/表单；单条直达表单 |
| 交期评审 · 选方案/报价单/订单 | 列表字段见 L3 §5.3 | 各来源取数与排序 | 选单 → 评审表单 |
| 交期评审 · 需求引导 | 需求输入；老客可跳过 | 新老客户规则同 02 | → 交期自选选品 |
| 订单选单卡（交期自选） | 选品推荐区、确认选品 | 同 02 选品规则 | → 评审表单 |
| 订单选单卡（复制/变更/进度） | 序号、订单号、日期金额、品项（无状态徽章） | 本客户历史订单倒序；复制/进度可筛需求 | 点选 / 第 N 条 |
| 交期评审 · 表单 | 来源摘要、期望交期、工艺版本表、采购计划 | 明细与工艺候选项 | 校验 → 提交 → 结果 |
| 交期评审 · 结果 | 徽章；按期一行说明+下单/调整；无法按时原因与参数摘要 | 评审结论与阻塞原因 | 下单 / 调整方案 / 仍要生成订单 |
| 复制订单 · 需求筛选 | 品项特征描述、可跳过 | 仅老客户 | → 选单；话术直达明细 |
| 复制订单 · 明细确认 | 手风琴改价、合计 | 来源订单明细 | 重选 / 确认复制 → 下单确认 |
| 订单变更 · 确认变更 | 订单摘要、受理说明 | 绑定订单 | 重选 / 确认变更 |
| 订单变更 · 变更已提交 | 订单号、受理说明 | — | 停留对话 |
| 订单进度 · 需求筛选 | 同复制需求逻辑 | 本客户历史订单 | → 选单 / 直达详情 |
| 订单进度 · 详情 | 订单头、宽表明细、时间轴 | 订单主档与节点 | 重选订单 |

详细卡片规格以功能需求说明 **§5** 为准；`annotation-spec-data.js` 与上表及验收面板同源。
