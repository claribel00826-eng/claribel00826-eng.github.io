# PRD · v1.5.0（增量 · 设计稿 v2）

> **基于**：[v1.4.0 PRD](../v1.4.0/PRD.md)  
> **状态**：阶段 A · **待用户确认**（本文档仅为设计，不含代码实现）  
> **本版范围**：① 最近访问模块优化（**以功能为主轴**） ② 新增客户

---

## 本版目标

在 v1.4.0 之上：**最近访问**改为记录并续接 **最近使用的功能**（库存、产能、报价等），客户仅作 **可选上下文**；补齐 **新增客户** 建档能力。

## 变更摘要（相对 v1.4.0）

| 维度 | v1.4.0 | v1.5.0（设计） |
|------|--------|----------------|
| 最近访问 | 以客户为主（客户名 + 技能）；静态 3 条 | **以功能为主**（功能名 + 进度）；客户为副信息；运行时写入 + 续接 |
| 新增客户 | 无 | 多入口建档；字段：编码/名称/性质/所属类别/地区 |
| 选客户可见范围 | 老客户 ∪ 公海新客户 | 增加 **我负责且无订单** 的客户（新建认领后可见） |

# 功能一 · 最近访问模块优化

## 1.1 设计原则（用户反馈）

> **不一定会选客户** —— 最近访问的重点是 **功能**，不是客户。

v1.4.0 中 12 项技能里有 **5 项不需客户**（今日待跟、产能分析、库存查询、业务分析、回款分析）。用户常直接点欢迎区或技能条进入全厂类能力，若以客户为主轴，这些访问 **无法正确表达**，也与真实使用习惯不符。

**本版主轴**：一条最近访问 = **一次功能使用记录**；`customerId` 降为 **可选附加上下文**。

## 1.2 展示规格

**位置**：首屏助手气泡，「待跟进摘要」下方，`data-spec-id="chat-recent"`。

**标题**：最近访问

**列表**：纵向卡片，最多 **5** 条，按 `visitedAt` 倒序。

**去重键**：`skillId`（同一功能只保留 **最新一条**；再次进入同一功能则更新时间与进度，不新增行）。

### 单条结构（功能优先）

**情形 A · 无客户上下文**（全厂类 / 未选客户即进入）

```
┌─────────────────────────────────────────┐
│ 库存查询                       10 分钟前 │  ← 主行：功能名 + 相对时间
│ 筛选：仅看可用为 0                       │  ← 次行：进度/筛选摘要（可选）
└─────────────────────────────────────────┘
```

**情形 B · 有客户上下文**（需客户类技能且当时已选客户）

```
┌─────────────────────────────────────────┐
│ 产品报价                          昨天   │
│ 华东精密机械 · 选品中                    │  ← 次行：客户名（副）· 进度摘要
└─────────────────────────────────────────┘
```

| 层级 | 字段 | 规则 |
|------|------|------|
| 主行 | 功能名 | 技能中文名（与欢迎区/技能条一致） |
| 主行右 | 相对时间 | 由 `visitedAt` 动态计算 |
| 次行 | 进度摘要 | 各技能上报，≤16 字；无则省略整行次行 |
| 次行前缀 | 客户名 | **仅当** `customerId` 有值且客户仍可解析时展示，后接「 · 」 |

**不展示**：老客户/新客户角标（避免把模块拉回客户导向）；客户名超长省略。

**空态**：「暂无最近访问，使用下方功能后将出现在此。」

## 1.3 写入时机

**前提**：当前企业上下文已就绪（已登录、已选企业）。**不要求**已选客户。

| 序号 | 触发节点 | 记录内容 |
|------|----------|----------|
| 1 | 用户通过欢迎区 / 技能条 / 话术 **进入某技能**且弹出首卡 | `skillId`；`customerId` = 当时顶栏客户（可为 null）；`checkpoint` = 入口 |
| 2 | 技能内产生 **有效进度**（选品、筛选、填槽、切 Tab 等） | 更新 `checkpoint` + `checkpointLabel` + `visitedAt` |
| 3 | 全厂类技能变更 **筛选/视图状态**（如库存「仅看可用为 0」） | 更新 `checkpointLabel` 为筛选摘要 |
| 4 | 点击最近访问续接成功 | 刷新 `visitedAt` |

**不单独写入**：仅切换客户、未进入任何技能；仅切换企业。

**演示种子**：`DemoData.recentVisits`（替代原 `recentCustomers`）示例：

```javascript
[
  { skillId: 'inventory', customerId: null, checkpointLabel: '全量概览', label: '10 分钟前' },
  { skillId: 'quote', customerId: 'c1', checkpointLabel: '选品中', label: '昨天' },
  { skillId: 'capacity', customerId: null, checkpointLabel: '甘特图', label: '3 天前' }
]
```

**存储**：`localStorage` · `sc-recent-visits:{enterpriseId}:{salesUser}` · 按 `skillId` upsert · 截断 5 条。

## 1.4 进度摘要（各技能示例）

| 技能 | needsCustomer | checkpointLabel 示例 |
|------|---------------|----------------------|
| 今日待跟 | 否 | 待跟进列表 / 客户详情 |
| 产能分析 | 否 | 甘特图 / 订单详情 |
| 库存查询 | 否 | 全量概览 / 仅看可用为 0 |
| 业务分析 | 否 | 汇总看板 |
| 回款分析 | 否 | 月度明细 |
| 方案速配 | 是 | 待填需求 / 选品中 / 待确认方案 |
| 产品报价 | 是 | 待填需求 / 选品中 / 待确认报价 |
| 交期/下单/复制/变更/进度 | 是 | 按 v1.4 主步骤映射 |

## 1.5 点击续接

```
点击某条最近访问（skillId + 可选 customerId + checkpoint）
  → switchActiveSkill(skillId)
  → 若该技能 needsCustomer：
        ├─ 记录含 customerId 且客户仍可见 → switchCustomer(customerId)
        ├─ 记录无 customerId → 不切换客户；技能内走现有「请先选客户」引导
        └─ 记录含 customerId 但客户已不可见 → 清空客户 + 技能内引导重选
  → 有 flowDraft/checkpoint → Skills.resume(skillId, checkpoint)
  → 无草稿 → 模拟话术 + Skills.run(skillId)
  → 更新 visitedAt
```

**与 v1.4.0 差异**：不再 **强制** `switchCustomer`；全厂类技能 **零客户依赖** 即可一键回到上次视图。

## 1.6 不在本版范围

- 跨设备同步、手动删单条、按客户筛选最近访问

# 功能二 · 新增客户

## 2.1 目标

对话流内快速建档；字段对齐 ERP 主数据最小集；成功后自动设为当前客户。

## 2.2 入口规划（含建议补充）

### 已确定入口

| 优先级 | 入口 | 说明 |
|--------|------|------|
| P0 | 选客户 inline 卡底部 | 「+ 新增客户」，与列表同框，意图最明确 |
| P0 | 话术 | 「新增客户 / 添加客户 / 建档 / 创建客户」→ 直接出表单卡 |
| P1 | 搜索无结果 | 列表空且搜索词非空 →「未找到匹配客户」+「新建客户」（名称预填搜索词） |

### 建议补充入口（推荐采纳）

| 优先级 | 入口 | 理由 |
|--------|------|------|
| **P0** | **缺客户引导卡** | 需客户技能在未选客户时弹出的引导卡（「选择客户」旁）增加 **「新建客户」**。与业务动作 **同上下文**，转化率高 |

### 不建议放置

| 入口 | 原因 |
|------|------|
| **顶栏「当前客户」区** | **v1.5 明确不做**；主框架保持 v1.4，避免顶栏操作过载 |
| 欢迎区 6×2 功能格 | 业务入口，非主数据 |
| 底部技能条 | 同上 |
| 待跟进摘要 | 语境不对 |

**推荐最小集（实现）**：缺客户引导卡 + 选客户卡底部 + 搜无结果 + 话术。

## 2.3 表单字段（用户指定）

**形态**：对话内 inline 卡 · `data-spec-id="card-customer-create"` · 标题「新增客户」

| 字段 | 必填 | 控件 | 规则 |
|------|------|------|------|
| **编码** | — | 只读展示 | **系统自动编码**；打开表单即生成预览号（如 `C-NEW-20260619-001`），提交后落库 |
| **名称** | ✓ | 单行输入 | 2～80 字；同企业完全同名禁止提交 |
| **性质** | — | 只读 | 固定 **「客户」**（对应档案 `partnerType = customer`），**不可修改** |
| **所属类别** | ✓ | **分级联动** | 见 §2.4；**一级必选，二级选填**（可只提交一级） |
| **地区** | ✓ | **分级选择** | 见 §2.5；国内到 **区/县**；国际须选到 **城市** |

**移除**（相对 v1 设计稿）：联系人、联系电话、需求简述 —— 本版表单 **不含** 这些字段；联系信息可在后续「写跟进」或详情维护中补全。

**操作**：提交 · 取消

## 2.4 所属类别 · 一级 / 二级（草案）

演示数据需与 v1.4 `categoryProductTypes` **一级对齐**，二级为行业/渠道细分（**请确认或替换为真实 ERP 枚举**）：

| 一级 | 二级（示例，均可不选） |
|------|--------------|
| 终端客户 | 制造业 / 装备业 / 科技服务 / 其他 |
| 经销商 | 区域经销 / 行业经销 / 电商渠道 / 其他 |
| 战略客户 | 集团总部 / 重点工程 / 其他 |

**交互**：

- **一级必选**；**二级选填** —— 允许只选一级即提交，**不强制**选二级。
- 一级未选 → 二级 disabled；一级变更 → 清空二级。
- 二级 placeholder：「可不选」。

**落库**：`category` = 一级（必有）；`categoryL2` = 二级或空；详情展示「终端客户」或「终端客户 · 制造业」。

**页面方案**：见 [ui-design-spec.md](./ui-design-spec.md) §7.4。

## 2.5 地区 · 分级选择（草案）

**第一步 · 范围**：`国内` | `国际`（单选，默认国内）

**国内**（三级联动，颗粒度到 **区/县**）：

```
省 / 直辖市  →  市  →  区/县
```

- 演示态使用静态省市区数据（精简版即可）；选完后写入 `regionScope: 'domestic'`，`regionProvince`，`regionCity`，`regionDistrict`，展示用 `regionLabel`（如「江苏省 · 苏州市 · 工业园区」）。

**国际**（须选到 **城市**）：

```
国家  →  州/省  →  城市
```

- 三项 **均必选**；写入 `regionScope: 'international'`，`regionCountry`，`regionState`，`regionCity`，`regionLabel`（如「德国 · 巴伐利亚 · 慕尼黑」）。

**校验**：国内须选满省市区；国际须选满国家、州/省、城市。

**页面方案**：见 [ui-design-spec.md](./ui-design-spec.md) §7.5。

## 2.6 提交后数据契约

```javascript

  id: 'c-local-{timestamp}',
  enterpriseId: state.enterpriseId,
  code: 'C-NEW-{yyyyMMdd}-{seq}',   // 自动编码

  partnerType: 'customer',          // 性质固定：客户
  category,                         // 所属类别一级
  categoryL2,                       // 所属类别二级

  regionCountry, regionState,                       // 国际

  customerType: 'new',
  level: '—',
  lastOrderAt: '—',
  lastOrderAmount: '—',
  accountManager: DemoData.demoSalesUser,
  latestFollowStatus: 'pending',
  contactName: '', contactPhone: '',  // 本版表单不填，留空
  updatedAt: ISO,
  _localCreated: true

```

## 2.7 业务规则

| 规则 | 说明 |
|------|------|
| 企业归属 | 当前顶栏企业 |
| 创建即认领 | `accountManager` = 当前业务员 |
| 可见性 | 选客户列表扩展：**我负责且无订单** |
| 重名 | 同企业名称完全相等 → 禁止提交 |
| 成功后续 | 系统提示 → 设为当前客户 → **下一步卡** |

## 2.8 不在本版范围

- 编辑/作废客户、附件、审批流、对接真实 ERP 接口

# 数据契约确认清单
