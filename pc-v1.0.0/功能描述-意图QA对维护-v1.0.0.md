# 功能描述 · 意图识别 QA 对维护（PC v1.0.0）

> **读者**：产品、NLU/前端研发、测试  
> **关联**：H5 `v1.5.0/QA对-主功能口语映射-v1.5.0.xlsx`、`scripts/build-intent-qa.py`、`js/intent-match.js`（方案 C）

---

## 1. 背景与目标

### 1.1 现状

- 主功能口语映射以 **Excel** 为 SSOT，经 Python 脚本生成 `intent-qa.generated.js`。
- H5 运行时由 `IntentMatch.matchMainFunctions()` 做 L0 召回（方案 C：多 Q 命中取 A 并集）。
- Excel 手工维护成本高：缺校验预览、歧义 Q 难发现、版本难对比。

### 1.2 本功能目标

在 **PC 管理端** 提供 **L0 主功能识别** QA 对的 **可视化维护、校验、试跑、导出**，使运营/算法同学不必直接改 Excel。

**维护边界**：用户说法 Q → 主功能 A（14 个枚举）。**不维护**卡片级子路由、槽位填槽、交互模式判定。

### 1.3 不做什么（v1.0.0）

- **不维护子卡片 / 卡片级路由**（如「交期异常详情」「选客户」等具体卡片跳转，见 `意图识别说明-v1.3.0.md` 六列表；规划 v1.2.0 另开模块）。
- 不维护槽位填槽规则（`annotation-docs/04-数据填槽需求.md`）。
- 不接真实登录/权限体系（演示级：本地单用户）。
- 不替代 `intent-smoke.mjs` 全量回归（仅提供快捷试跑入口）。

## 4. 数据模型

### 4.1 QA 对字段（与 Excel sheet「QA对」对齐）

| 字段 | 键名 | 必填 | 说明 |
|------|------|------|------|
| 序号 | `id` | 否 | 行号/自增，导入时可空 |
| 用户说法 Q | `q` | 是 | 口语问句，去空白后参与匹配 |
| 目标功能 A | `a` | 是 | **枚举**，见 §4.2 |
| 备注 | `note` | 否 | 如「写跟进归入今日待跟」 |
| 状态 | `status` | 否 | `draft` 草稿 / `published` 已上线 / `offline` 已下线；新建默认草稿，不可选手动状态 |

### 4.2 目标功能 A 枚举（与 `build-intent-qa.py` 一致）

| 显示名 | aId |
|--------|-----|
| 今日待跟 | followup |
| 新增客户 | customer-create |
| 方案速配 | plan |
| 产品报价 | quote |
| 交期评审 | delivery |
| 确认下单 | order |
| 复制订单 | copy |
| 订单变更 | change |
| 订单进度 | progress |
| 产能分析 | capacity |
| 库存查询 | inventory |
| 业务分析 | biz-analysis |
| 回款分析 | payment |
| 切换客户 | switch-customer |

### 4.3 派生数据（只读，不手改）

| 结构 | 说明 |
|------|------|
| `multiQ[]` | 同一 Q 对应 ≥2 个不同 A 的索引，供歧义页展示 |
| 统计 | `pairCount`、`uniqueQCount`、各功能 A 分布 |

### 4.4 版本包（工作区）

```json
{
  "version": "v1.5.0",
  "algorithm": "C",
  "updatedAt": "ISO8601",
  "pairs": [ /* QA 对数组 */ ],
  "multiQ": [ /* 自动生成 */ ],
  "featureIds": { /* 枚举表 */ }
}
```

PC 端本地默认读写：`pc-v1.0.0/data/qa-workspace.json`；导出 Excel 文件名约定：`QA对-主功能口语映射-{version}.xlsx`。

## 5. 页面设计

### 5.1 布局（PC 宽屏）

```
┌──────────────────────────────────────────────────────────────────┐
│ 顶栏：品牌  │  试跑验证 │ 导入 Excel │ 导出 Excel │ 新增 │
├────────────┬─────────────────────────────────────────────────────┤
│ 侧栏导航   │  筛选条：搜索 Q · 主功能 A · 状态 · 仅歧义 Q        │
│ 主功能 QA  │  数据概览：QA 对 / 唯一 Q / 歧义 Q（列表上方）       │
│            │  QA 对列表（表格，每页 10 条）                       │
│            │  ┌──┬──┬──────────┬────────┬──────────┬────┬───┐   │
│            │  │□│# │ Q        │ A      │ 备注     │状态│操作│   │
│            │  └──┴──┴──────────┴────────┴──────────┴────┴───┘   │
│            │  分页 · 共 N 条 · 已选 M 条                        │
└────────────┴─────────────────────────────────────────────────────┘

试跑验证：弹窗（非底栏/内嵌），输入话术 → 展示 L0 命中结果
```

- 页面背景 `#F5F5F5`，内容区 padding 24px。
- 主色 `#108482`（与 H5 演示一致）。
- 表格行高 48px，卡片圆角 8px。

### 5.2 QA 对列表

| 列 | 说明 |
|----|------|
| 多选 | 批量删除 |
| 序号 | 当前列表连续序号；列头可排序（默认 ↑ 激活） |
| Q | 主列，可点击进编辑；列头可排序 |
| A | 功能标签；列头可排序 |
| 备注 | 超长省略，hover 全文；由运营手工填写，非系统生成 |
| 状态 | 草稿 / 已上线 / 已下线（标签展示） |
| 操作 | 编辑、复制、删除（已上线不可删）、试跑 |

**状态流转**：新建 → 草稿 → **上线** → 已上线；已上线须 **下线** 后才可删除；已下线可 **上线** 恢复。

**分页**：每页 10 条。勾选「仅歧义 Q」时按 Q 再按 A 排序以便相邻对照。

**默认排序**：按序号（id）升序，序号列头 ↑ 高亮。序号 / 用户说法 Q / 目标主功能 A 三列均常显 ↑↓，点击列头切换排序字段与升/降序。

### 5.3 编辑抽屉（单条 / 双击行）

| 字段 | 控件 |
|------|------|
| Q | 多行输入，实时字数 |
| A | 下拉（§4.2 枚举） |
| 备注 | 多行 |

新建无状态选择，保存为草稿。抽屉底部：**保存**（草稿）、**上线**（草稿/已下线/新建）；已上线显示 **下线**；草稿/已下线可 **删除**。

**校验（保存时）**：

- Q、A 非空
- A 必须在枚举内
- 与已有行 `(q, a)` 完全重复 → 阻止保存并提示行号

保存后：更新 workspace JSON，重算 multi-Q。

### 5.4 歧义 Q 索引页（Tab 或侧栏入口）

列表仅展示 **multi-Q**（每页 10 条）：

| 列 | 说明 |
|----|------|
| Q | |
| 命中 A 列表 | 如 `交期评审`、`产能分析`（标签展示） |
| 操作 | 跳转 QA 列表并筛选该 Q |

用途：运营确认是否为 **合理多意图**；不合理则改 Q 或删冗余行。

### 5.5 试跑验证（弹窗）

- 点击页头「试跑验证」或列表行内「试跑」打开弹窗。
- 输入框 +「试跑」。
- 调用与 H5 相同算法（内嵌 `intent-match.js` 逻辑或 PC 端同构实现）。
- 展示：
  - 归一化后文本
  - 命中功能列表（name + aId + note）
  - 若 0 命中 / 多命中，给出提示文案

可选：**从列表行一键填入试跑框并自动试跑**。

### 5.6 导入 / 导出

| 操作 | 行为 |
|------|------|
| 导入 Excel | 打开导入弹窗：下载模板 + 拖拽/选择文件上传 |
| 导出 Excel | 页头按钮；生成 `QA对-主功能口语映射-{version}.xlsx`，可直接覆盖 H5 源文件 |

编辑/导入/删除后自动写入浏览器 localStorage；JS 产物由研发侧 `build-intent-qa.py` 生成。

## 6. 核心流程

### 6.1 日常维护

```
打开 PC 端 → 加载 workspace / 从 Excel 导入
  → 列表筛选编辑
  → 导出 Excel
  →（研发）v1.5.0 下运行 build-intent-qa.py
  → H5 验证 intent-smoke
```

### 6.2 与 H5 构建链

| 步骤 | 负责方 |
|------|--------|
| PC 维护 QA 对 | 运营/产品 |
| 导出 Excel 至 `v1.5.0/` | PC 端或人工拷贝 |
| `python v1.5.0/scripts/build-intent-qa.py` | 研发 CI 或本地 |
| `node v1.5.0/scripts/intent-smoke.mjs` | 测试 |

PC v1.0.0 **可内置**「一键构建」按钮（调用同级 repo 内脚本），但 v1.0.0 不强制。

## 7. 校验规则（与构建脚本一致）

| 规则 | 处理 |
|------|------|
| Q 或 A 为空 | 禁止保存/导出 |
| 未知 A | 禁止保存 |
| 完全重复 (q, a) | 导入时跳过；编辑时阻止 |
| 同 Q 多 A | 允许（方案 C）；列入 multi-Q 索引 |
| Q 仅空白差异 | 归一化后视为同 Q（与 `normalize()` 一致） |

归一化规则（与 `build-intent-qa.py` / `intent-match.js` 一致）：

- 去首尾空白、去所有空格
- 去标点：`，,。.；;！!？?、`

## 12. 启动方式（v1.0.0 已实现）

本地静态服务（仓库根目录）：

```bash
npx --yes serve -l 3456 -c serve.json
```

浏览器打开：`http://localhost:3456/pc-v1.0.0/index.html`（或 `/pc-v1.0.0/`，需带 `-c serve.json`）

- 首次加载 `data/qa-workspace.json`（155 条，来自 H5 `intent-qa.generated.js`）
- 编辑后自动写入 localStorage
- 「导出 Excel」生成与 H5 构建脚本兼容的 xlsx

同步种子数据：

```bash
node pc-v1.0.0/scripts/sync-from-v15.mjs
```
