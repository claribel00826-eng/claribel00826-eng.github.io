# 功能描述 · 意图识别 QA 对维护（PC v1.0.0）

> **读者**：产品、NLU/前端研发、测试  
> **关联**：H5 `v1.5.0/QA对-主功能口语映射-v1.5.0.xlsx`、`scripts/build-intent-qa.py`、`js/intent-match.js`（方案 C）

---

## 1. 背景与目标

### 1.1 现状

- 主功能口语映射以 **Excel** 为 SSOT，经 Python 脚本生成 `intent-qa.generated.js`。
- H5 运行时由 `IntentMatch.matchMainFunctions()` 做 L0 召回（方案 C：多 Q 命中取 A 并集）。
- Excel 手工维护成本高：缺校验预览、歧义 Q 难发现、版本难对比。

### 1.2 本功能目标

在 **PC 管理端** 提供 QA 对的 **可视化维护、校验、试跑、导出**，使运营/算法同学不必直接改 Excel。

### 1.3 不做什么（v1.0.0）

- 不维护卡片级路由表（见 `意图识别说明-v1.3.0.md` 六列表）。
- 不维护槽位填槽规则（`annotation-docs/04-数据填槽需求.md`）。
- 不接真实登录/权限体系（演示级：本地单用户）。
- 不替代 `intent-smoke.mjs` 全量回归（仅提供快捷试跑入口）。

## 4. 数据模型

### 4.1 QA 对字段（与 Excel sheet「QA对」对齐）

| 字段 | 键名 | 必填 | 说明 |
|------|------|------|------|
| 序号 | `id` | 否 | 行号/自增，导入时可空 |
| 用户说法 Q | `q` | 是 | 口语问句，去空白后参与匹配 |
| 目标功能 A | `a` | 是 | **枚举**，见 §4.2 |
| 分组 | `group` | 否 | 如 `待跟进-入口`，列表筛选与批量操作 |
| 备注 | `note` | 否 | 如「写跟进归入今日待跟」 |
| 状态 | `status` | 否 | v1.0.0：`draft` / `published`（默认 published） |

### 4.2 目标功能 A 枚举（与 `build-intent-qa.py` 一致）

| 显示名 | aId |
|--------|-----|
| 今日待跟 | followup |
| 新增客户 | customer-create |
| 方案速配 | plan |
| 产品报价 | quote |
| 交期评审 | delivery |
| 确认下单 | order |
| 复制订单 | copy |
| 订单变更 | change |
| 订单进度 | progress |
| 产能分析 | capacity |
| 库存查询 | inventory |
| 业务分析 | biz-analysis |
| 回款分析 | payment |
| 切换客户 | switch-customer |

### 4.3 派生数据（只读，不手改）

| 结构 | 说明 |
|------|------|
| `multiQ[]` | 同一 Q 对应 ≥2 个不同 A 的索引，供歧义页展示 |
| 统计 | `pairCount`、`uniqueQCount`、各功能 A 分布 |

### 4.4 版本包（工作区）

```json
{
  "version": "v1.5.0",
  "algorithm": "C",
  "updatedAt": "ISO8601",
  "pairs": [ /* QA 对数组 */ ],
  "multiQ": [ /* 自动生成 */ ],
  "featureIds": { /* 枚举表 */ }
}
```

PC 端本地默认读写：`pc-v1.0.0/data/qa-workspace.json`；导出 Excel 文件名约定：`QA对-主功能口语映射-{version}.xlsx`。

## 5. 页面设计

### 5.1 布局（PC 宽屏）

```
┌──────────────────────────────────────────────────────────────────┐
│ 顶栏：版本选择 v1.5.0 ▼  │  已保存 / 有未发布改动  │  导出 ▼   │
├────────────┬─────────────────────────────────────────────────────┤
│ 左侧筛选   │  QA 对列表（表格）                                   │
│ · 功能 A   │  ┌────┬──────────┬────────┬──────────┬────┬───┐ │
│ · 分组     │  │ □ │ Q        │ A      │ 分组     │备注│ ⋮ │ │
│ · 状态     │  ├────┼──────────┼────────┼──────────┼────┼───┤ │
│ · 仅歧义 Q │  │   │ 查交期   │ 交期评审│ 交期-入口│    │   │ │
│ · 搜索 Q   │  └────┴──────────┴────────┴──────────┴────┴───┘ │
│            │  分页 · 共 N 条 · 已选 M 条                        │
├────────────┴─────────────────────────────────────────────────────┤
│ 底栏试跑： [ 输入用户话术________________ ] [试跑]  → 命中: 交期评审 │
└──────────────────────────────────────────────────────────────────┘
```

- 页面背景 `#F5F5F5`，内容区 padding 24px。
- 主色 `#108482`（与 H5 演示一致）。
- 表格行高 48px，卡片圆角 8px。

### 5.2 QA 对列表

| 列 | 说明 |
|----|------|
| 多选 | 批量删除 / 批量改分组 / 批量改 A |
| Q | 主列，可点击进编辑 |
| A | 功能标签（色块按功能区分） |
| 分组 | |
| 备注 | 超长省略，hover 全文 |
| 歧义 | 若 Q 在 multi-Q 中，显示 ⚠ |
| 操作 | 编辑、复制为新行、删除 |

**默认排序**：分组 → A → Q 拼音。

### 5.3 编辑抽屉（单条 / 双击行）

| 字段 | 控件 |
|------|------|
| Q | 多行输入，实时字数 |
| A | 下拉（§4.2 枚举） |
| 分组 | 输入 + 历史分组联想 |
| 备注 | 多行 |
| 状态 | 草稿 / 已发布 |

**校验（保存时）**：

- Q、A 非空
- A 必须在枚举内
- 与已有行 `(q, a)` 完全重复 → 阻止保存并提示行号

保存后：更新 workspace JSON，重算 multi-Q。

### 5.4 歧义 Q 索引页（Tab 或侧栏入口）

列表仅展示 **multi-Q**：

| 列 | 说明 |
|----|------|
| Q | |
| 命中 A 列表 | 如 `交期评审、产能分析` |
| 操作 | 跳转筛选该 Q 的全部 QA 行 |

用途：运营确认是否为 **合理多意图**；不合理则改 Q 或删冗余行。

### 5.5 试跑面板

- 输入框 +「试跑」。
- 调用与 H5 相同算法（内嵌 `intent-match.js` 逻辑或 PC 端同构实现）。
- 展示：
  - 归一化后文本
  - 命中功能列表（name + aId + group + note）
  - 若 0 命中 / 多命中，给出提示文案

可选：**从列表选中一行 Q 一键填入试跑框**。

### 5.6 导入 / 导出

| 操作 | 行为 |
|------|------|
| 导入 Excel | 解析 sheet「QA对」，列顺序与现网一致；导入前预览差异（新增/变更/删除） |
| 导出 Excel | 生成 `QA对-主功能口语映射-{version}.xlsx`，可直接覆盖 H5 源文件 |
| 导出 JSON | 导出 workspace，便于 Git diff |
| 生成 JS 预览 | 调用构建逻辑，预览 `intent-qa.generated.js` 片段（不直接写 H5 目录，需确认路径后写入） |

**写入 H5 确认弹窗**：目标 `v1.5.0/js/intent-qa.generated.js`，显示 pairCount / multi_q 统计。

## 6. 核心流程

### 6.1 日常维护

```
打开 PC 端 → 加载 workspace / 从 Excel 导入
  → 列表筛选编辑
  → 试跑验证
  → 保存 workspace
  → 导出 Excel
  →（研发）v1.5.0 下运行 build-intent-qa.py
  → H5 验证 intent-smoke
```

### 6.2 与 H5 构建链

| 步骤 | 负责方 |
|------|--------|
| PC 维护 QA 对 | 运营/产品 |
| 导出 Excel 至 `v1.5.0/` | PC 端或人工拷贝 |
| `python v1.5.0/scripts/build-intent-qa.py` | 研发 CI 或本地 |
| `node v1.5.0/scripts/intent-smoke.mjs` | 测试 |

PC v1.0.0 **可内置**「一键构建」按钮（调用同级 repo 内脚本），但 v1.0.0 不强制。

## 7. 校验规则（与构建脚本一致）

| 规则 | 处理 |
|------|------|
| Q 或 A 为空 | 禁止保存/导出 |
| 未知 A | 禁止保存 |
| 完全重复 (q, a) | 导入时跳过；编辑时阻止 |
| 同 Q 多 A | 允许（方案 C）；列入 multi-Q 索引 |
| Q 仅空白差异 | 归一化后视为同 Q（与 `normalize()` 一致） |

归一化规则（与 `build-intent-qa.py` / `intent-match.js` 一致）：

- 去首尾空白、去所有空格
- 去标点：`，,。.；;！!？?、`
